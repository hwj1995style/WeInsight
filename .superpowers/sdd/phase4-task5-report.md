## 异步请求与依赖边界

### 默认依赖

`create_app` 增加可注入的 `report_request_service` 和 `report_request_repo`。缺省构建时复用 Web 已创建的同一个 SQLAlchemy Engine，并构建：

- `MysqlReportRequestRepo`；
- Task 3 `ReportGenerationService`；
- 群/文章日报生成 Service；
- 复用既有 summary 查询 Service 的只读汇总生成依赖。

测试确认请求 Repo、请求 Service、两类日报生成 Repo 和三类查询 Repo 均共享同一 Engine。Web 不调用 `execute_request`，也不持有 Worker 或数据库连接。

### 手动请求

GET `/reports` 使用 `secrets.token_urlsafe(32)` 为每次渲染生成不可预测、规范长度的 hidden idempotency key。同一已渲染表单重复提交继续携带同一个 key，由 Task 3 Repo 保证幂等。

POST `/reports/generate` 严格要求且只允许五个单值字段：

- `csrf_token`
- `idempotency_key`
- `report_type`
- `report_date`
- `source_name`

未知字段、重复字段、缺失字段、非法 key/type/date/source，以及 summary/all 携带 source，均返回固定安全 422。认证由现有中间件统一执行，CSRF 同时经过中间件和路由常量时间复核；actor 固定使用当前认证管理员用户名。请求通过 `run_in_threadpool` 调用 `request_manual`，成功后 303 到 `/reports/requests/{id}`。

未来日期、幂等冲突和意外数据库错误均映射为固定安全页面，不回显 SQL、路径、密码、异常类型或输入详情。

## 请求状态与 HTMX

GET `/reports/requests/{request_id}` 只接受正整数 ID，通过线程池调用 Repo；不存在或非法 ID 返回安全 404。页面显示请求类型、日期、对象、提交人、cutoff、创建/开始/结束时间和安全错误摘要，不显示 worker ID 或 lease。

状态文案：

- pending：等待处理；
- running：生成中；
- success：生成成功；
- partial_success：部分成功；
- failed：生成失败。

仅 pending/running 的状态容器带 `hx-get` 和 `hx-trigger="every 2s"`，并从本地静态资源加载 HTMX 2.0.4。终态响应完全移除轮询属性，因此替换后自动停止轮询。

## Markdown 与下载安全

### 安全渲染

renderer 固定 allowlist 为：h1-h6、p、ul、ol、li、strong、em、code、pre、table、thead、tbody、tr、th、td、blockquote。

处理顺序：

1. HTML parser 丢弃 script/style/iframe/object/embed/svg/math/template 及其内容；
2. 丢弃 HTML 注释和全部原始标签/属性；
3. 移除 javascript/data 协议标记；
4. MarkdownIt 以 `html=False`、`linkify=False` 渲染；
5. Bleach 使用固定标签、空 attributes、空 protocols 再清洗；
6. 只将最终清洗结果包装为 `Markup`。

测试覆盖嵌套和畸形 HTML、脚本内容、事件/style 属性、注释、javascript/data、图片、链接、表格和非字符串输入。

### 下载

固定下载路由：

- `/reports/group/{date}/{source}.md`
- `/reports/article/{date}/{source}.md`
- `/reports/summary/{date}.md`

群和文章下载只调用对应 QueryService `get_report`，返回数据库中的 `markdown_body`。汇总下载只调用 `load_sources`，使用两类已落库摘要中的对象、标题和计数字段组装 Markdown；不会拼接两类正文。汇总单元格处理换行、反斜杠、pipe 和尖括号，避免破坏表格或注入 HTML。

三类响应统一设置：

- `Content-Type: text/markdown; charset=utf-8`
- `Content-Disposition: attachment`
- `X-Content-Type-Options: nosniff`

文件名为固定 ASCII 类型加日期，不把 source 放入 header。date/source 严格校验并拒绝点路径、斜杠、控制字符和 CRLF；不存在返回安全 404，服务异常返回固定 503。
