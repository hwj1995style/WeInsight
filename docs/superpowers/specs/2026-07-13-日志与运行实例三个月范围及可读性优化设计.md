# 日志与运行实例三个月范围及可读性优化设计

## 1. 背景与现状

管理后台的“日志”和“运行实例”当前允许浏览全部历史数据。随着运行记录增长，列表分页和筛选会持续扫描旧数据；同时结构化日志直接展示英文事件类型、内部阶段、完整 Worker ID、脱敏后的技术消息和 JSON 指标，运维人员难以快速判断“发生了什么、是否成功、影响了谁”。

本次改造只影响“日志”和“运行实例”两个页面及其关联访问入口，不限制任务列表，不删除数据库历史数据。

## 2. 目标与非目标

### 2.1 目标

- 日志与运行实例只允许查看当前时刻向前滚动 3 个日历月的数据。
- 上海时区 2026-07-13 的可查看起点为 2026-04-13；使用日历月运算，不使用固定 90 天。
- 列表、筛选、分页、运行详情、运行详情中的事件和日志实时流使用同一时间边界。
- 直接访问超期运行实例时，返回“该记录已超出可查看范围”，与“运行实例不存在”区分。
- 日志页面采用“运维摘要表”：中文业务结果和关键数字优先，技术字段按需展开。
- 未识别的新事件类型安全降级，不影响页面加载。

### 2.2 非目标

- 不删除、归档或迁移三个月前的数据库记录。
- 不限制任务列表、任务详情和 Worker 心跳页面。
- 不改变采集、清洗、分析、报价提取和事件写入逻辑。
- 不修改日志级别、事件类型或指标 JSON 的存储格式。

## 3. 影响范围

- 后端运行监控查询：运行列表、运行详情、结构化日志列表和日志实时流。
- Web 路由：`/runs`、`/runs/{id}`、`/events`、`/events/stream`。
- 前端模板：运行列表范围说明、日志筛选区和日志摘要表。
- 样式：日志级别标签、指标胶囊、可展开技术详情及移动端布局。
- 测试：日期边界、查询约束、分页、筛选、详情访问、实时流和语义化展示。

## 4. 数据模型与权限模型

不新增表和字段，不执行历史数据清理。

所有时间判断使用应用统一的上海时区时钟。边界函数返回 `now - relativedelta(months=3)` 的等价日历月结果，并转换为数据库查询使用的无时区上海本地时间。月末采用合法日期回退规则，例如 5 月 31 日向前 3 个月为 2 月最后一天。

权限规则沿用现有管理员会话与 CSRF 规则。本次只收窄读取范围，不新增角色或授权能力。即使管理员知道超期运行 ID，也不能通过详情页或事件流读取超期内容。

## 5. 后端实现方案

### 5.1 统一时间边界

在运行监控服务建立单一“最早可查看时间”计算入口，由可注入时钟产生。路由不得分别使用 `datetime.now()`，避免列表、详情和实时流边界漂移，也便于确定性测试。

### 5.2 查询层限制

- 运行实例列表在分页前增加 `scheduled_at >= :visible_since`。
- 日志列表在全部筛选条件和分页前增加 `create_time >= :visible_since`。
- 运行详情先按 ID 查询，再区分：不存在、存在但 `scheduled_at < visible_since`、正常可见。
- 运行详情最近事件和 `/events/stream` 同时约束事件时间和所属运行的可见范围，不能通过 `after_id` 绕过时间限制。
- 页面参数不得放宽三个月边界；用户传入更早开始时间时，后端取其与 `visible_since` 的较晚值。

### 5.3 日志语义化视图模型

后端将安全事件转换为只读展示模型，不修改原始事件：

- `summary`：中文运维摘要，例如“本轮公众号采集完成”“公众号处理完成”“开始采集”。
- `subject`：任务、运行或公众号目标等关联对象。
- `severity`：由现有 level 映射的视觉等级。
- `metric_items`：按固定白名单和顺序输出中文指标，包括目标、成功、失败、新增、重复、跳过、读取。
- `technical_details`：原始事件类型、阶段、Worker ID 和已安全截断的指标摘要。

已知事件使用显式映射表；未知事件显示“未分类事件”，保留安全的原始事件类型作为技术详情。消息继续经过现有脱敏和长度限制，模板不得直接解析或执行原始 JSON。

## 6. 前端设计

### 6.1 日志运维摘要表

表格主列调整为：

- 时间：上海时间和事件 ID。
- 结果摘要：中文标题、关联对象及状态标签。
- 关联对象：任务、运行和目标链接。
- 关键指标：中文指标胶囊，只显示存在的指标。
- 操作：`<details>` 展开的“技术详情”。

Worker ID、英文事件类型、内部阶段和原始指标摘要默认折叠。INFO 使用普通样式，WARN 与 ERROR 使用醒目标识，但保持方向 A 的蓝灰视觉体系。

筛选区继续保留任务、运行、目标、链路和级别筛选，并明确显示“仅展示最近 3 个月”。空结果文案区分“范围内暂无日志”和参数错误。

### 6.2 运行实例

运行实例列表保持现有布局，只增加“仅展示最近 3 个月”的范围说明。分页总数和页面条数以过滤后的集合为准。

直接访问超期详情时使用独立提示页或标准错误面板，文案固定为“该记录已超出可查看范围”，并提供返回运行实例列表的链接。

### 6.3 响应式与可访问性

- 桌面端保持摘要表横向滚动能力。
- 移动端优先展示时间、摘要和状态；技术详情在行内展开。
- 状态不只依赖颜色，必须有中文文字。
- `<details>` 使用原生键盘可访问交互。

## 7. 错误处理

- 超期运行详情使用稳定的业务错误，不泄露该记录的任务名称、Worker、错误或指标。
- 非法筛选参数继续返回现有安全提示。
- 未知事件、非法指标或指标过大时使用现有安全摘要，并显示“指标不可用”或“指标已截断”。
- 实时流请求超期运行 ID 时立即拒绝，不建立持续连接。

## 8. 测试与回归方案

### 8.1 时间边界

- 普通日期向前 3 个日历月。
- 月末回退、2 月和闰年。
- 边界时刻等于起点可见，早 1 微秒不可见。
- 全部计算以上海时区为准。

### 8.2 查询与访问控制

- 运行列表和日志列表在分页前应用时间条件。
- 现有任务、运行、级别等筛选与时间边界同时生效。
- 用户传入更早时间不能放宽范围。
- 超期运行详情显示固定提示；不存在仍显示“不存在”。
- 运行详情事件和 SSE 不能读取超期运行或事件。

### 8.3 日志展示

- 已知事件映射为中文摘要。
- 指标白名单、中文标签、顺序和缺失值处理正确。
- WARN/ERROR 状态文字可见。
- 未知事件安全降级。
- Worker、原始事件类型、阶段和指标默认位于技术详情中。
- 桌面端与移动端浏览器截图复核，无控制台错误。

### 8.4 全量回归

运行完整 pytest，确认任务管理、采集调度、状态页、Worker 和报告页面不受影响。

## 9. 风险与分阶段落地建议

- 风险：日期边界在月末产生歧义。通过统一日历月函数和边界测试控制。
- 风险：列表已限制但详情或 SSE 漏限。通过服务层统一授权入口与直接 URL 测试控制。
- 风险：事件映射遗漏导致空白。未知事件必须降级为“未分类事件”。
- 风险：技术详情过长影响页面。继续使用现有脱敏、深度、条数和字节限制。

建议一次交付但分三步验证：先落统一查询边界，再落日志展示模型，最后进行桌面端与移动端视觉验收。历史数据保留，因此回滚只需恢复查询与模板，不涉及数据恢复。
