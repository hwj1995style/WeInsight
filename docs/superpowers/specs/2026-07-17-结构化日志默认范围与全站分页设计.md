# 结构化日志默认范围与全站分页设计

## 1. 背景与现状

结构化日志页目前提供“包含正常轮次”复选框，默认不勾选；用户确认应删除该筛选并默认展示正常轮次。现有分页在日志、任务、运行、结果、来源和日报页面各自实现，大多只有上一页和下一页，无法直接回到首页、跳至尾页或输入指定页。

排查 2026-07-17 日志中的密集 ERROR 后确认：这些记录每 5 分钟出现一次，事件类型均为 `pipeline_stage_failed`，阶段均为 `werss_authorization_monitor`。WeRSS 返回及 MySQL `DATETIME` 读取的授权到期时间可能为无时区 `datetime`，Pipeline Worker 使用 `Asia/Shanghai` 有时区 `datetime`，两者在到期状态计算中相减触发 `TypeError`。WeRSS 状态接口、管理凭据和邮件配置读取均正常。用户随后明确要求修复并清理这些周期性 ERROR，因此本设计追加时区修复与精准清理方案。

## 2. 目标与非目标

### 2.1 目标

1. 日志页删除“包含正常轮次”控件，列表固定包含正常轮次。
2. 全部列表分页统一支持首页、上一页、指定页、下一页、尾页。
3. 跳页时保留当前筛选条件和每页数量。
4. 页码输入严格限制为正整数且不超过尾页，服务端继续承担最终校验。
5. 分页在桌面端保持紧凑，在移动端可换行且不横向溢出。
6. 授权监控进入状态计算前，将上游及数据库中的到期时间统一归一化为 `Asia/Shanghai` 有时区时间。
7. 仅清理由该缺陷产生的历史周期性 ERROR，保留其他错误和审计记录。

### 2.2 非目标

1. 不改变各列表的默认每页数量、数据排序和保留周期。
2. 不引入前端框架或客户端分页。
3. 不清理其他阶段、其他异常类型或无法确认由本缺陷产生的 ERROR。

## 3. 影响范围

- 日志查询：移除可变的正常轮次过滤，固定 `include_routine=True`。
- 公共前端：新增统一分页模板与样式。
- 列表路由：为日志、任务、运行、结果、来源和日报提供统一分页上下文。
- 数据查询：来源和日报列表补充总数查询，用于可靠计算尾页。
- 授权监控：在服务层统一归一化到期时间，覆盖实时上游状态和数据库历史状态两条路径。
- 历史数据：按事件类型、阶段、异常类型、空业务关联和安全消息等条件精准删除缺陷噪声。
- 测试：覆盖筛选兼容、分页 URL、指定页表单、总页数和响应式布局。

## 4. 数据与权限模型

不新增数据表，不改变权限。分页参数继续使用 GET 查询参数：`page` 为从 1 开始的正整数，`page_size` 沿用各页面现有上限。指定页表单以隐藏字段保留当前筛选参数，避免丢失查询上下文。

旧书签中的 `include_routine` 参数继续允许进入，但不再改变结果，避免历史链接直接返回 422；页面生成的新 URL 不再携带该参数。

## 5. 后端实现方案

新增统一分页构建器，根据路径、筛选参数、当前页、每页数量和总记录数生成：

- `total_pages`
- 首页、上一页、下一页、尾页 URL
- 指定页表单 action 与需保留的查询参数

事件、任务、运行和结果查询已返回 `total_count`，直接复用。来源仓库新增轻量 `COUNT(*)`；日报查询仓库按日期和可选对象新增计数接口。汇总日报的尾页按微信群与公众号两类总数的较大值计算，与当前并排分页语义一致。

请求页码大于尾页时仍由现有服务返回空页，不做隐式重定向；分页控件给出正确尾页，用户可直接返回。

授权服务新增统一时间归一化函数：无时区时间按 WeRSS 和本系统共同使用的上海本地时间解释并附加 `Asia/Shanghai`；已有时区时间转换到 `Asia/Shanghai`。状态判断、剩余时间展示、快照版本生成均使用归一化后的时间，避免实时刷新成功和读取历史快照时再次发生混算。

历史清理在事务中执行，删除条件必须同时满足：`level='error'`、`event_type='pipeline_stage_failed'`、`stage='werss_authorization_monitor'`、结构化异常类型为 `TypeError`、无任务/运行/目标关联、Worker 事件及该阶段固定安全消息。清理前后分别统计数量与时间范围，禁止按 `ERROR` 级别或阶段单条件扩大删除范围。

## 6. 前端实现方案

统一分页区域分为三组：

1. 左侧记录信息：`第 X / Y 页，共 N 条`。
2. 中间跳页表单：数字输入框和“跳转”按钮。
3. 右侧导航：首页、上一页、下一页、尾页；不可用状态使用禁用文本，不产生无效链接。

移动端三组内容换行，跳页输入保持足够触控尺寸。所有分页使用明确的 `aria-label`，指定页输入带最大页约束。

## 7. 测试与回归方案

1. 日志页面不存在“包含正常轮次”，服务调用固定 `include_routine=True`。
2. 旧 `include_routine` 参数不会改变固定包含策略。
3. 每个分页页面渲染首页、尾页、指定页表单及正确的总页数。
4. 筛选参数在首页、尾页和跳页隐藏字段中保持不变。
5. 来源与日报计数 SQL 使用绑定参数并与列表过滤一致。
6. 运行全量测试并使用 Playwright 验证日志页首尾页、指定页跳转以及移动端布局。
7. 使用有时区的当前时间分别覆盖无时区上游到期时间、无时区数据库快照和已有时区到期时间。
8. 在本地实际执行一次授权监控，确认状态快照成功更新且不新增 `werss_authorization_monitor` ERROR。
9. 清理前后核对精准条件命中数，确认其他 ERROR 数量不变。

## 8. 风险与分阶段建议

| 风险 | 控制措施 |
| --- | --- |
| 各页面筛选参数丢失 | 分页上下文统一从已校验的标准化查询值生成 |
| 总数与列表过滤不一致 | 计数查询复用相同日期、来源和可见范围条件 |
| 输入超大页码造成昂贵 OFFSET | 页码输入前端限制尾页，服务端保留正整数与页大小校验 |
| 移动端控件拥挤 | 分组换行，按钮使用紧凑变体，跳页输入限制宽度 |
| 无时区时间语义不明确 | 按 WeRSS 部署约定将其解释为 `Asia/Shanghai` 本地时间，并在服务边界统一归一化 |
| 历史清理误删真实故障 | 使用事件类型、阶段、异常类型、关联字段、Actor 和固定安全消息联合约束，事务内核对 |

## 9. 验收标准

1. 日志页默认可见正常轮次且不再展示相关筛选控件。
2. 所有已有分页页面均可跳首页、尾页和任意合法页。
3. 跳页后当前筛选和每页数量不丢失。
4. 桌面端与 390px 移动端无分页布局溢出。
5. 全量自动化测试通过，现有列表查询与权限无回归。
6. 授权监控连续执行不再产生时区 `TypeError`，授权状态及到期提醒正常计算。
7. 历史周期性噪声已清理，其他 ERROR 和正常轮次日志保留。
