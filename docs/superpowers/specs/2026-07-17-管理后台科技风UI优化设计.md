# WeInsight 管理后台科技风 UI 优化设计

## 1. 背景与现状

现有管理后台已经具备深色侧栏、浅色工作区和统一面板，但视觉仍偏传统企业后台：侧栏与内容区的品牌联系较弱，运行状态主要依赖绿色，ECharts 成功序列和报价矩阵存在大面积绿色，面板、表格、筛选器和数字指标的层级不够鲜明。

本次优化在不改变业务流程的前提下，强化 WeInsight 作为数据采集与运行监控平台的技术感和控制台气质，并响应“尽量少出现绿色”的视觉要求。

## 2. 目标与非目标

### 2.1 目标

- 统一为深靛蓝应用壳、冷白蓝灰工作区、电蓝与克制紫色高光的科技控制台风格。
- 保留现有导航顺序、页面信息架构、数据密度和操作路径。
- 将绿色从主视觉和图表中移除；正常、在线、成功默认使用蓝色或青蓝色表达。
- 绿色仅作为兼容性极小语义点使用，且必须同时包含文字状态，不能单独依赖颜色。
- 增强侧栏、工具栏、KPI、表格、筛选器、表单、对话框和图表的一致性。
- 保持桌面、平板和手机的可读性、键盘可访问性和现有交互能力。

### 2.2 非目标

- 不修改后端接口、路由、数据模型、权限模型、数据库或 Worker 行为。
- 不引入 React、Vue、前端构建链或外部 CDN。
- 不新增业务模块、指标、导航、登录方式或虚构数据。
- 不使用高强度霓虹、玻璃拟态、装饰插画或影响可读性的内容覆盖渐变。
- 不将宽表格改造成会丢失字段关系的卡片列表。

## 3. 影响范围

- 全局应用壳：`base.html`、`app-shell.js`、`app.css`。
- 登录页：`login.html`。
- 实时总览：`dashboard.html` 及两张 ECharts 图表主题。
- 共享页面组件：列表、表格、筛选器、表单、按钮、状态、分页、对话框、空状态。
- 报价矩阵：移除绿色表头、网格、焦点与阴影，替换为冷蓝体系。
- 浏览器缓存：更新 CSS 静态资源版本参数。

## 4. 数据模型与权限模型

- 数据模型无变化。
- Session、CSRF、登录限流、密码校验和路由权限无变化。
- 所有状态继续由现有模板变量和 ECharts 数据驱动。
- 颜色只作为辅助表达；在线、失败、警告、停用等状态必须保留文字或图标。

## 5. 视觉设计系统

### 5.1 色彩令牌

- 侧栏背景：`#071a35`，深层背景：`#051225`。
- 工作区：`#f3f7fd`，面板：`#ffffff`，弱面板：`#f7faff`。
- 主文字：`#10213d`，次级文字：`#667796`。
- 主强调色：`#1468f3`，悬浮：`#0c56d7`，焦点：`#45a6ff`。
- 次强调色：`#7758ff`，用于次要数据序列和细微重点，不用于大面积底色。
- 成功/在线：默认电蓝 `#1677ff`；错误 `#d14343`；警告 `#c47a12`；未知/停用 `#8b98ae`。
- 兼容绿色：仅保留在极小语义点或第三方不可控内容中，禁止用于图表主序列、按钮、焦点、面板和报价矩阵。

### 5.2 字体与数字

- 中文与界面字体继续使用 `Inter`、`Segoe UI`、`PingFang SC`、`Microsoft YaHei` 系统栈。
- 数字和时间启用 `font-variant-numeric: tabular-nums`。
- 页面标题强调紧凑字距和明确层级；控件、导航和表格显式定义字号与行高。

### 5.3 几何、边框与阴影

- 控件圆角 7px，面板圆角 10px，避免过度胶囊化。
- 使用冷蓝 1px 边框和克制阴影；选中态使用左侧或顶部 2px 高光轨道。
- 工作区使用低对比度技术网格背景，只提供空间质感，不覆盖内容。
- 侧栏可使用 CSS 线性网格/电路线条背景，强度必须低于导航文字。

### 5.4 组件与动效

- 侧栏选中态采用蓝色边框、冷蓝填充和轻微外发光。
- KPI 使用开放式分隔结构，避免面板内继续堆叠卡片。
- 状态使用小圆点、细轨和文字，成功与在线不再使用绿色胶囊。
- 表头使用冷蓝灰背景，行悬浮使用极浅蓝色。
- 动效仅用于侧栏、按钮、面板悬浮和抽屉，时长 160–220ms，并尊重 `prefers-reduced-motion`。

## 6. 页面设计

### 6.1 登录页

- 深靛蓝技术纹理背景承载居中的双栏登录壳。
- 左侧显示品牌、标题、说明和代码原生 W 标记；右侧保留管理员登录表单。
- 输入框使用青蓝焦点环，登录按钮使用电蓝，不出现绿色。
- 移动端保持单列，品牌区压缩但保留标题和视觉标记。

### 6.2 实时总览

- 保持现有系统运行、业务概览、双图表和待处理任务顺序。
- 系统运行与业务概览使用细蓝色左轨和竖向数据分隔，增强扫描效率。
- 在线、正常、成功状态改用蓝色点与蓝色文字；失败继续使用红色，警告使用橙色。
- 两张图表的成功序列改用蓝色；次要序列使用紫色或中性灰；不再使用绿色。
- 图表 tooltip、坐标轴、网格线、图例和柱体圆角与全局令牌一致。

### 6.3 列表、表单和报价矩阵

- 列表与筛选器统一冷白面板、蓝色焦点和弱蓝行悬浮。
- 主按钮使用电蓝，次按钮使用淡蓝底，危险按钮继续使用红色语义。
- 报价矩阵的表头、固定列、焦点框和滚动阴影改为蓝灰体系；推算值继续使用琥珀色。
- 对话框与移动抽屉使用深靛蓝遮罩，不使用绿色状态面板。

## 7. 后端实现方案

- 后端不需要修改。
- 路由、模板上下文、表单字段、CSRF、会话和数据库访问保持不变。
- 若静态资源缓存影响验收，仅更新模板中的 CSS 版本参数。

## 8. 前端实现方案

- 在 `app.css` 中集中更新令牌并补充科技网格、状态点、数据轨道和组件状态。
- 保留现有 class 和模板结构，避免大范围结构重写。
- 在 `base.html` 和 `login.html` 中补充代码原生品牌标记与必要语义 class。
- 在 `dashboard.html` 中仅调整 ECharts 配色和视觉参数，不改变数据绑定、无脚本回退与 aria 描述。
- 所有图形元素使用现有 SVG 或 CSS，不引入外部图片依赖。

## 9. 测试与回归方案

### 9.1 自动化测试

- 新增科技风令牌与禁用绿色主序列的静态约束测试。
- 验证登录页和应用壳包含品牌技术标记与资源版本。
- 验证 ECharts 成功序列使用蓝色，模板中不再出现旧绿色图表色值。
- 保持现有 Web、鉴权、来源、任务、运行、日志、结果和日报测试通过。
- 运行 `git diff --check` 和全量 `pytest -q`。

### 9.2 浏览器视觉与交互验证

- Playwright 验证登录、总览和代表性表格页。
- 桌面视口：1440×900；移动视口：390×844。
- 验证登录、侧栏展开/折叠、导航、图表加载、表格滚动和移动抽屉。
- 截图与概念图逐项比较：信息架构、首屏层级、色彩、字体、间距、边框、图表和状态表达。
- 检查控制台 warning/error、横向溢出、键盘焦点和 `prefers-reduced-motion`。

## 10. 风险与分阶段落地建议

### 10.1 风险

- `app.css` 为全站共享文件，令牌修改可能影响旧页面的对比度和状态语义。
- 绿色替换后，用户可能短期内不习惯蓝色表示成功，需要依靠文字与状态点维持清晰度。
- 报价矩阵、授权状态和图表存在局部硬编码颜色，遗漏会破坏统一性。
- CSS 背景纹理必须足够克制，避免降低小字号表格可读性。

### 10.2 分阶段落地

1. 建立新令牌与静态约束测试。
2. 优化应用壳、登录页和共享组件。
3. 优化总览状态与图表主题。
4. 清理报价矩阵及其他硬编码绿色。
5. 完成桌面、移动和自动化回归。

## 11. 验收标准

- 后台整体具有一致、克制、可读的科技控制台风格。
- 主视觉、图表、焦点、按钮和报价矩阵不使用绿色。
- 正常、在线和成功状态由蓝色与文字共同表达。
- 现有路由、权限、数据、表单和主要交互无回归。
- 桌面与移动视口无内容裁切、异常换行或横向页面溢出。
- Playwright 最终截图与概念图在信息架构、色彩、层级和密度上达到可交付一致性。
