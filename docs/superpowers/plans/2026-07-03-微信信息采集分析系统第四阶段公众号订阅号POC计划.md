# 微信信息采集分析系统第四阶段公众号订阅号 POC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不影响微信群链路稳定试运行的前提下，逐步落地公众号/订阅号链路的受控 POC 能力。

**Architecture:** 第四阶段只围绕 article 链路推进，继续保持 group/article 两套配置、任务、日志、进度和安全输出隔离。真实微信 UI 操作仍必须通过 `wechat_ui_lock` 串行交错执行；所有 article POC 命令默认先走配置、fake RPA、入库、解析和恢复验证，不直接开启生产真实采集。

**Tech Stack:** Python 3.12, pytest, SQLAlchemy/PyMySQL, MySQL 8.4, PowerShell, Markdown runbook.

## 执行原则

1. 先补设计、迁移说明和运行说明，再实现命令。
2. 所有新增命令必须先写失败测试，再实现。
3. article 链路不得读写 group 任务、游标、日志和日报。
4. 公众号/订阅号真实采集开关默认保持关闭。
5. 第四阶段每一步完成后都必须运行专项测试和全量测试。
6. 开发环境继续不注册 Windows 计划任务。

## Task 29: 真实 UI 拿链接 POC

- [x] 进入订阅号入口、定位账号、复制当天文章链接。
- [x] 获取链接后立即释放 `wechat_ui_lock`。
- [x] 失败保存截图和错误码，不重试占用 UI。
- [x] 默认命令要求显式传入账号名，禁止自动扫全部账号。

落地说明：

1. `app/rpa/wxauto_client.py` 新增 `WxautoArticleRpaClient`，复用 wxauto/wxauto4 初始化逻辑，通过 `ChatWith(account_name)` 打开指定公众号/订阅号，并从可见消息或卡片文本中提取 `mp.weixin.qq.com/s/` 链接。

4. `app/security/output_policy.py` 已将 `collect-article-once` 纳入安全输出白名单，只允许输出账号名、计数和 `rpa_error`。
5. 本任务只实现受控真实 UI 拿链接入口；完整标题、发布时间和正文解析放到 Task 30。

## Task 30: Playwright 文章解析链路

- [x] 新增文章链接解析服务，使用浏览器打开链接，提取标题、发布时间、摘要和正文长度。
- [x] 解析阶段不持有 `wechat_ui_lock`。
- [x] 解析失败只标记 article 任务失败，不影响微信群链路。
- [x] CLI、日志和日报不输出大段原文。

落地说明：

1. `sql/init.sql` 新增 `wechat_article_clean`，只保存解析后的标题、发布时间、作者、摘要、正文长度和解析时间，不保存大段正文。
2. `sql/migrations/20260706_002_create_article_clean.sql` 用于已有库增量建表，迁移只创建 article clean 表，不访问微信群表。
3. `app/domain/article_parsing.py` 定义 article 解析源、解析结果和 clean 记录。
4. `app/pipelines/article_parse_service.py` 提供 `ArticleParseService`、HTML 元数据提取器和 `PlaywrightArticleParser`。服务不接收 UI lock 依赖。
5. `app/storage/article_parse_repo.py` 只消费 `wechat_article_process_task(task_type='clean_article')`，成功后写入 `wechat_article_clean`、创建 `analyze_article` 任务并标记 clean 任务成功；失败只更新 article 任务 retry/status/error。
6. `app.main` 新增 `parse-article-once`，不做微信健康检查，不打开微信 UI，只输出 read/success/failed 计数。
7. `app/security/output_policy.py` 已将 article 正文相关内部字段加入敏感输出防线，并将 `parse-article-once` 纳入安全输出白名单。
