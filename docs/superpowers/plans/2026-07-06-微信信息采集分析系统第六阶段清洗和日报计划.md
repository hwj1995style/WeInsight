# 微信信息采集分析系统第六阶段清洗和日报 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不增加微信 UI 风险的前提下，补齐公众号/订阅号文章分析、文章日报和可选汇总日报只读聚合层。

**Architecture:** 微信群链路的清洗、分析和群日报已经在前置阶段完成，阶段六不重做群链路。阶段六聚焦 article 后处理：运行时临时读取正文，落库只保存摘要和结构化特征，不保存正文或 HTML；article 服务只写 article 自己的分析表、日报表和任务状态。汇总日报只读 group/article 日报结果，不回写任一采集链路状态。所有清洗、分析、日报和汇总命令均不获取 `wechat_ui_lock`，不打开微信 PC UI。

**Tech Stack:** Python 3.12, pytest, SQLAlchemy/PyMySQL, MySQL 8.4, Markdown reports, PowerShell.

---

## 设计检查清单

背景与现状：

1. 群消息 raw -> clean -> analysis -> group daily report 链路已具备手动运行、查看、导出和安全输出回归。
2. 公众号/订阅号 raw -> clean 链路已完成，`wechat_article_clean` 只保存文章元数据、摘要和正文长度，不保存大段正文。
3. `wechat_article_process_task` 已支持 `clean_article/analyze_article/article_daily_report` 任务类型，但 `analyze_article` 和 `article_daily_report` 尚未落地。

目标：

1. 补齐 article 分析结果表、文章基础摘要和标签分析。
2. 补齐 article 日报表、生成服务、查看和导出 CLI。
3. 明确汇总日报只读聚合层设计，保证失败不回写 group/article 链路状态。

非目标：

1. 不接入 AI 分析；阶段七再做 AI 灰度。
2. 不保存正文或 HTML；运行时临时读取正文，落库只保存摘要和结构化特征。
3. 不注册 Windows 计划任务。
4. 不打开微信 UI，不获取 `wechat_ui_lock`。

影响范围：

1. SQL schema 和 migrations 新增 article analysis / article daily report 表。
2. 后端新增 article analysis、article daily report 服务和仓储。
3. CLI 新增 article 日报查看和导出命令。
4. README 和运行手册补充 article 后处理命令。

数据模型：

1. `wechat_article_analysis` 按 `article_hash` 唯一保存分析结果，只保存摘要、主题标签、关键词命中、正文长度和分析版本等结构化字段。
2. `wechat_article_daily_report` 按 `report_date + account_name` 唯一保存文章日报。
3. 汇总日报只读 `wechat_group_daily_report` 和 `wechat_article_daily_report`。

权限模型：

1. article 命令不得读写微信群任务、游标、日志和日报状态。
2. group 命令不得读写 article 任务、分析和日报状态。
3. 汇总日报命令只读，不更新 group/article 任一任务表。

测试与回归：

1. 每个任务先写失败测试，再实现。
2. 增加 schema、migration、domain、service、repo、CLI、安全输出和隔离回归测试。
3. 每项任务至少运行目标测试；阶段收口运行 `pytest -q`。

风险与分阶段落地：

1. 文章摘要质量先采用规则摘要，不调用外部 AI，避免合规和成本风险。
2. 文章日报只读 clean/analysis 结构化字段，不输出正文或链接。
3. 汇总日报先以设计和只读边界为主，确认 group/article 独立日报稳定后再实现聚合命令。
4. 正文只允许在 Playwright/analysis 进程内存中短暂存在；异常日志只记录错误摘要和任务 ID，不记录正文片段。

Add a small deterministic helper:

```python
def extract_price_items(extracted_tables: list[dict]) -> list[dict]:
    items: list[dict] = []
    for table in extracted_tables:
        headers = table.get("headers", [])
        for row in table.get("rows", []):
            row_map = dict(zip(headers, row, strict=False))
            if "含包装价" in row_map:
                items.append(
                    {
                        "source_media_type": table.get("source_media_type", "html_table"),
                        "title": table.get("title", ""),
                        "spec": row_map.get("规格", ""),
                        "weight": row_map.get("毛重", ""),
                        "price": row_map.get("含包装价", ""),
                        "change": row_map.get("涨", ""),
                    }
                )
            if "今日价" in row_map:
                items.append(
                    {
                        "source_media_type": table.get("source_media_type", "image_ocr"),
                        "title": table.get("title", ""),
                        "weight": row_map.get("净重", ""),
                        "yesterday_price": row_map.get("昨日价", ""),
                        "today_price": row_map.get("今日价", ""),
                        "change": row_map.get("涨跌", ""),
                    }
                )
    return items
```
