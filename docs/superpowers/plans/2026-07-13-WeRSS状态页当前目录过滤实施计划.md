# WeRSS 状态页当前目录过滤实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 公众号状态页只展示当前 WeRSS 目录中的启用和停用来源。

**Architecture:** 在 `MysqlArticleSourceStatusRepo` 的主查询中、分页之前增加来源范围条件。数据库历史记录及采集、清洗、分析数据保持不变，Web 与服务层不做二次过滤。

**Tech Stack:** Python 3.12、SQLAlchemy、MySQL 8、pytest。

## Global Constraints

- 仅展示 `werss_source_id IS NOT NULL` 且 `upstream_status IN ('active', 'disabled')` 的记录。
- WeRSS 中停用的来源继续展示。
- `missing`、`excluded`、`unknown` 不展示，但不得删除数据库历史。
- 过滤必须发生在 SQL 的 `ORDER BY`、`LIMIT` 和 `OFFSET` 之前。

**Interfaces:**

- Produces: 相同返回接口，但结果只含当前 WeRSS 的 `active/disabled` 来源。

- [ ] **Step 1: 写 SQL 契约失败测试**

WHERE config.werss_source_id IS NOT NULL
