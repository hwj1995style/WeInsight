# WeInsight 管理后台阶段一 Web 鉴权基础 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立可独立运行和测试的 FastAPI 管理后台基础，支持默认管理员、数据库 Session、登录锁定、CSRF、主动改密和默认密码持续提醒。

**Architecture:** FastAPI 应用通过 `app.state` 注入 AuthService；管理员和 Session 保存在 MySQL，浏览器只保存 session token 与 CSRF token。Web 基础与真实 RPA 完全隔离，本阶段不创建采集任务、不启动 Worker。

**Tech Stack:** Python 3.12、FastAPI 0.115.12、Uvicorn 0.34.0、Jinja2 3.1.6、python-multipart 0.0.32、argon2-cffi 23.1.0、httpx 0.28.1、SQLAlchemy 2.0.32、PyMySQL 1.1.1、pytest 8.3.2、MySQL。

## Global Constraints

- 设计基线：`docs/superpowers/specs/2026-07-10-微信采集管理后台设计.md`。
- 本阶段不导入 `app.rpa`，不启动采集 Worker，不获取 `wechat_ui_lock`。
- 默认账号固定为 `admin/admin123456`；首次登录不强制改密。
- 默认密码未修改时，所有已登录页面必须持续显示安全提醒。
- 密码只保存 Argon2id 哈希；Session 和 CSRF 令牌在数据库中只保存 SHA-256 哈希。
- 连续登录失败 5 次锁定 15 分钟。
- 生产 Cookie 必须使用 `HttpOnly`、`Secure`、`SameSite=Strict`；CSRF Cookie 不设置 HttpOnly，以便 HTMX 读取，但仍使用 `Secure` 和 `SameSite=Strict`。
- 所有非安全 HTTP 方法必须校验 CSRF；登录 POST 使用独立的登录表单 token，不依赖已登录 Session。
- 不引入 Redis、Celery、Node 构建链或第三方身份服务。
- 每个任务结束后运行局部测试和完整 `pytest -q`。

- [ ] **Step 3: 实现密码与令牌函数**

`app/security/passwords.py` 暴露：

```python
from argon2 import PasswordHasher as Argon2PasswordHasher
from argon2.exceptions import InvalidHashError, VerificationError, VerifyMismatchError


class PasswordHasher:
    def __init__(self) -> None:
        self._hasher = Argon2PasswordHasher()

    def hash(self, password: str) -> str:
        return self._hasher.hash(password)

    def verify(self, password_hash: str, password: str) -> bool:
        try:
            return self._hasher.verify(password_hash, password)
        except (VerifyMismatchError, VerificationError, InvalidHashError):
            return False
```

令牌统一使用 `secrets.token_urlsafe(32)`，哈希统一使用 `hashlib.sha256(value.encode("utf-8")).hexdigest()`。

```python

find_user_by_id(user_id: int) -> AdminUserRecord | None
create_user_if_missing(username: str, password_hash: str) -> bool
record_login_failure(

```

所有 SQL 使用绑定参数；`find_active_session` 同时校验 `revoked_at IS NULL`、`expires_at > :now`、`idle_expires_at > :now` 和用户 enabled。
`find_user_by_id` 供主动改密验证当前密码使用；损坏或不受支持的 Argon2 哈希必须按验证失败处理，不得抛出为 500 错误。
`create_user_if_missing` 必须使用单条原子 `INSERT ... SELECT ... WHERE NOT EXISTS`，仅当管理员表中不存在任何记录时创建默认管理员；用户名唯一键只作为并发兜底。
`record_login_failure` 必须在同一数据库事务内锁定用户行，依据数据库当前计数完成自增与阈值判断，返回更新后的 `locked_until`，且不得以空值覆盖已有有效锁定。
`update_password_and_revoke_sessions` 必须在同一数据库事务内更新密码并注销该管理员的全部 Session，任一 SQL 失败都要整体回滚。
`login` 对不存在或已禁用的管理员也必须执行一次与真实密码相同参数的 Argon2id dummy 哈希校验，再返回统一失败，避免通过响应耗时枚举用户名；已进入有效锁定期的账号继续直接拒绝，不重复消耗哈希计算。

- [ ] **Step 5: 实现 AuthService 精确接口**

`app/services/auth_service.py` 定义：

```python
DEFAULT_ADMIN_PASSWORD = "admin123456"


@dataclass(frozen=True)
class AuthenticatedAdmin:
    id: int
    username: str
    using_default_password: bool


@dataclass(frozen=True)
class AuthenticatedSession:
    admin: AuthenticatedAdmin
    session_token: str
    csrf_token: str
    expires_at: datetime


class InvalidCredentialsError(RuntimeError):
    pass


class LoginLockedError(RuntimeError):
    def __init__(self, locked_until: datetime) -> None:
        super().__init__(f"login locked until {locked_until.isoformat()}")
        self.locked_until = locked_until


class PasswordValidationError(ValueError):
    pass


```

`AuthService` 的精确方法签名为：

- `ensure_bootstrap_admin(self) -> None`
- `login(self, username: str, password: str, client_ip: str, user_agent: str, now: datetime) -> AuthenticatedSession`
- `authenticate(self, session_token: str, csrf_token: str | None, now: datetime) -> AuthenticatedAdmin | None`
- `verify_csrf(self, session_token: str, csrf_token: str, now: datetime) -> bool`
- `logout(self, session_token: str, now: datetime) -> None`
- `change_password(self, admin_id: int, current_password: str, new_password: str, now: datetime) -> None`

`change_password` 要求新密码至少 12 个字符，成功后通过 `update_password_and_revoke_sessions` 原子设置 `password_changed_at=now` 并注销该用户全部 Session。

- [ ] **Step 3: 实现应用工厂与中间件**

`app/web/app.py` 的公开入口固定为：

```python
def create_app(config: Config, auth_service: AuthService | None = None) -> FastAPI:
    app = FastAPI(title="WeInsight Admin", docs_url=None, redoc_url=None)
    app.state.config = config
    app.state.auth_service = auth_service or _build_auth_service(config)
    app.add_middleware(AdminSessionMiddleware)
    app.mount("/static", StaticFiles(directory="app/web/static"), name="static")
    app.include_router(auth.router)
    return app
```

`AdminSessionMiddleware` 必须：

1. 放行 `GET /login`、`POST /login`、`/static/*`、`GET /healthz`。
2. 从配置的 Session Cookie 读取 token 并调用 `authenticate`。
3. HTML 未登录请求返回 303 `/login`，JSON/SSE 未登录请求返回 401。
4. 对 POST/PUT/PATCH/DELETE 比对表单或 `X-CSRF-Token` 与 CSRF Cookie，并调用 `verify_csrf`。
5. 将 `request.state.admin` 和 `request.state.csrf_token` 注入模板。

畸形 URL 编码或 multipart 表单必须在鉴权中间件中安全返回 403，不得让解析异常越过 FastAPI 异常处理形成 500；非表单类异常继续向上抛出。

- [ ] **Step 4: 实现登录、退出和主动改密路由**

路由行为固定为：`GET /login` 渲染登录页，`POST /login` 接收 username/password/login_csrf，`POST /logout` 注销，`GET /account/password` 渲染主动改密页，`POST /account/password` 接收 current_password/new_password。

GET `/login` 设置 20 分钟有效的 `login_csrf` Cookie；POST `/login` 使用 `secrets.compare_digest` 校验表单值。登录成功设置 Session Cookie 和 CSRF Cookie；退出撤销数据库 Session 并删除两个 Cookie。

def test_web_entrypoint_builds_app_without_importing_rpa(monkeypatch) -> None:
    imported = []
    real_import = builtins.__import__

    def tracked_import(name, *args, **kwargs):
        imported.append(name)
        return real_import(name, *args, **kwargs)

    monkeypatch.setattr(builtins, "__import__", tracked_import)
    from app.web.app import create_app
    assert callable(create_app)
    assert not any(name.startswith("app.rpa") for name in imported)
```

def main() -> None:
    parser = argparse.ArgumentParser(prog="weinsight-web")
    parser.add_argument("--config", default="config/config.dev.yaml")
    args = parser.parse_args()
    config = load_config(Path(args.config))
    app = create_app(config)
    app.state.auth_service.ensure_bootstrap_admin()
    uvicorn.run(app, host=config.web.host, port=config.web.port)
```
