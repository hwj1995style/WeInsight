# 公众号蛋价规格转换与统一口径 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add first-version egg quote standardization so raw article quote rows can be compared on a `yuan_per_jin` basis.

**Architecture:** Extend the existing article egg price item model rather than creating a new aggregate table. Compute standardized price fields during `EggPriceItem` construction, persist them with the raw structured row, and mark only trustworthy chicken-egg rows as `include_in_standard_price`.

**Tech Stack:** Python 3.12, SQLAlchemy, PyMySQL, MySQL 8, pytest.

## Global Constraints

- Do not change the current no-OCR boundary.
- Do not touch `wechat_group_*` tables or group pipeline code.
- Do not overwrite original price, weight, specification, quote basis, or row context fields.
- First version standardizes to `yuan_per_jin` only.
- Chicken egg primary standard price uses `include_in_standard_price=1`; non-chicken products are not part of this primary口径.
- Plausible chicken egg `yuan_per_jin` range is `[2.0, 10.0]`; rows outside the range remain unconverted for primary口径.
