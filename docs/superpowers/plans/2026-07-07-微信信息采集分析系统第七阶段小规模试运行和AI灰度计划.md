# 微信信息采集分析系统第七阶段小规模试运行和AI灰度准备 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不扩大风险面的前提下，先完成无 AI 的小规模试运行、巡检和日报质量评估，再设计并灰度验证 AI 分析能力。

**Architecture:** 第七阶段分为两段：Task 51-53 只围绕现有 group/article/report 链路做试运行方案、巡检报告和日报质量评估；Task 54-55 再引入默认关闭的 AI 灰度能力；Task 56 统一复核生产配置和回滚预案。AI 灰度不得回写 group/article 原链路状态，必须使用独立配置、独立记录和可关闭开关。

**Tech Stack:** Python 3.12、pytest、MySQL 8.4、PowerShell、Markdown 运维文档、现有 `app.main` CLI、现有 `wechat_group_*` / `wechat_article_*` / 日报表。

## Global Constraints

1. 开发环境暂时不注册 Windows 计划任务。
2. 生产初版核心群不超过5个。
3. 生产初版公众号/订阅号不超过20个。
4. 微信 PC 版本以 4.1.8.107 为当前已验证版本。
5. 微信 UI 操作必须继续通过 `wechat_ui_lock` 串行互斥。
6. AI 默认关闭。
7. AI 灰度不发送 raw 原文、文章全文、HTML。
8. AI 失败不得回写 group/article 链路状态。
9. 任一阶段的巡检、导出和评估命令不得占用微信 UI，除非命令名称明确是采集调度。
10. 所有中文文档按 UTF-8 保存，并通过 `rg -n "\?\?" README.md docs app tests sql` 检查。

## 阶段七完成标准

1. 小规模试运行方案明确核心群不超过5个、公众号/订阅号不超过20个。
2. 试运行期间具备人工巡检报告或等效巡检命令。
3. 群日报、文章日报、汇总日报都有人工质量评估表。
4. AI 灰度设计明确默认关闭 AI。
5. AI 灰度不发送 raw 原文、文章全文、HTML。
6. AI 分析最小 POC 首版为 dry-run，不调用外部模型。
7. AI 失败不得回写 group/article 链路状态。
8. 生产配置和回滚预案可指导关闭 article 链路、关闭 AI、只保留群链路。
9. `pytest -q` 通过。
