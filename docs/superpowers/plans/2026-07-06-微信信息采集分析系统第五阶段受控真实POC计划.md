# 微信信息采集分析系统第五阶段受控真实 POC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不影响核心微信群链路的前提下，完成公众号/订阅号链路的人工值守真实 POC，并验证双链路隔离、UI 锁互斥、失败恢复和扩容边界。

**Architecture:** 阶段五继续沿用 group/article 双链路隔离。article 真实 POC 先从 1 个授权账号开始，只在人工命令下执行；微信 UI 操作仍通过 `wechat_ui_lock` 串行交错，解析和监控命令不持有 UI 锁。通过 1 个账号闭环和 3 个账号人工值守后，再评估是否进入最多 20 个账号的小规模试运行。

**Tech Stack:** Python 3.12, pytest, SQLAlchemy/PyMySQL, wxauto/wxauto4, Playwright, MySQL 8.4, PowerShell, Markdown runbook.

## 执行原则

1. 真实微信 UI 操作只允许发生在明确的人工 POC 命令中。
2. 开发阶段继续不注册 Windows 计划任务。
3. `article.enabled` 默认保持关闭；命令行手动执行不等于打开生产后台采集。
4. 公众号/订阅号链路一次 UI 锁只处理 1 个账号。
5. 核心群到期时，article 链路必须在安全检查点释放 UI 锁。
6. 任何 article 命令不得修改微信群配置、游标、任务、日志和日报。
7. 所有新增 CLI 必须先写失败测试，再实现命令。
8. 所有新增 CLI 输出必须纳入 `app/security/output_policy.py` 白名单。

- [x] **Step 3: Implement article task admin repo**

Create records and methods with this implementation shape:

```python
from __future__ import annotations

from dataclasses import dataclass
from datetime import datetime

from sqlalchemy import text
from sqlalchemy.engine import Engine


@dataclass(frozen=True)
class ArticleFailedTaskRecord:
    id: int
    task_type: str
    ref_type: str
    ref_id: str
    status: str
    retry_count: int
    next_run_time: datetime | None
    error_summary: str | None
    update_time: datetime | None


class MysqlArticleTaskAdminRepo:
    def __init__(self, engine: Engine) -> None:
        self.engine = engine

    def list_failed_tasks(self, task_type: str | None, limit: int) -> list[ArticleFailedTaskRecord]:
        statement = text(
            """
            SELECT
                id,
                task_type,
                ref_type,
                ref_id,
                status,
                retry_count,
                next_run_time,
                LEFT(COALESCE(error_msg, ''), 200) AS error_summary,
                update_time
            FROM wechat_article_process_task
            WHERE status = 'failed'
              AND (:task_type IS NULL OR task_type = :task_type)
            ORDER BY update_time DESC, id DESC
            LIMIT :limit
            """
        )
        with self.engine.begin() as connection:
            rows = connection.execute(statement, {"task_type": task_type, "limit": limit}).mappings().all()
        return [_row_to_failed_task(row) for row in rows]

    def retry_failed_tasks(self, task_type: str | None, limit: int) -> int:
        statement = text(
            """
            UPDATE wechat_article_process_task
            SET status = 'pending',
                retry_count = 0,
                next_run_time = CURRENT_TIMESTAMP,
                update_time = CURRENT_TIMESTAMP
            WHERE status = 'failed'
              AND (:task_type IS NULL OR task_type = :task_type)
            ORDER BY update_time DESC, id DESC
            LIMIT :limit
            """
        )
        with self.engine.begin() as connection:
            result = connection.execute(statement, {"task_type": task_type, "limit": limit})
        return int(result.rowcount or 0)


def _row_to_failed_task(row) -> ArticleFailedTaskRecord:
    return ArticleFailedTaskRecord(
        id=int(row["id"]),
        task_type=str(row["task_type"]),
        ref_type=str(row["ref_type"]),
        ref_id=str(row["ref_id"]),
        status=str(row["status"]),
        retry_count=int(row["retry_count"]),
        next_run_time=row["next_run_time"],
        error_summary=str(row["error_summary"] or "")[:200] or None,
        update_time=row["update_time"],
    )
```

`error_summary` must be truncated before returning.
