# 公众号报价矩阵展示实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将报价结果改为默认展示最新报价日、支持历史日期切换、按固定公众号规则生成并可解释缺码外推的“50–30 码 × 公众号”报价矩阵。

**Architecture:** 数据库仍保存逐条报价明细，安全查询层提供矩阵所需白名单原始行，独立的 `price_matrix` 领域模块负责公众号规则、文章决胜、区间展开、低高价拆列和最近价差外推。结果服务返回完整矩阵 DTO，FastAPI 路由与 Jinja 模板只负责日期输入和展示，不新增汇总表或人工规则配置入口。

**Tech Stack:** Python 3、FastAPI、Jinja2、SQLAlchemy 文本查询、MySQL、Decimal、pytest、Playwright、原生 CSS。

## Global Constraints

- 固定输出 50 至 30 码，按降序排列。
- 未指定日期时使用九个目标公众号数据中的最大报价日期。
- 同一公众号同日只使用发布时间最新的一篇；相同时间以文章哈希和记录 ID 稳定决胜。
- 河南金咕咕按元/斤展示，其余公众号按元/箱展示，不使用标准化价格覆盖原始价格。
- 价格存在上下限时拆分低价、高价列，低价列在前、高价列在后。
- 缺码使用距离最近的一组已知价差外推；距离相同时优先较小码数一侧。
- 原始值优先于推算值，推算值必须带可解释依据并在页面明显标识。
- 不跨公众号、日期或低高价列推算；只有一个已知值时保持空白。
- 只纳入当前 WeRSS 采集列表中的九个目标公众号。
- 页面不得输出文章链接、正文、原始 JSON 或其他敏感采集内容。
- 不新增数据库表，不修改采集、清洗、增量分析及十分钟调度流程。

@dataclass(frozen=True)
class PriceMatrixCell:
    value: Decimal | None
    source: Literal["observed", "extrapolated", "empty"]
    explanation: str | None = None

@dataclass(frozen=True)
class PriceMatrixColumn:
    key: str
    account_name: str
    label: str
    unit: str
    price_side: Literal["single", "low", "high"]

@dataclass(frozen=True)
class PriceMatrixRow:
    size: int
    cells: Mapping[str, PriceMatrixCell]

@dataclass(frozen=True)
class PriceMatrix:
    quote_date: date
    updated_at: datetime | None
    source_count: int
    columns: tuple[PriceMatrixColumn, ...]
    rows: tuple[PriceMatrixRow, ...]
    rules: tuple[AccountMatrixRule, ...]
