# 公众号 RSS 采集替换 RPA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变现有文章后处理结果模型的前提下，以通用 RSS/Atom 采集适配层完全替换公众号 RPA，并让微信群 RPA 与公众号 RSS 故障隔离。

**Architecture:** WeRSS 作为部署在当前 Windows 采集机 Docker Desktop 中的可替换 Feed 提供者；WeInsight 只通过 HTTP 条件请求读取标准 RSS/Atom。新的 Feed 客户端、条目映射器和采集服务把条目幂等写入 `wechat_article_raw`，现有解析、清洗、报价、分析和日报继续运行；微信群仍使用原 RPA 和 `wechat_ui_lock`。

**Tech Stack:** Python 3、FastAPI/Jinja2、SQLAlchemy/PyMySQL、httpx、feedparser、pytest、MySQL 8、Docker Desktop、WeRSS。

## Global Constraints

- WeRSS 部署在当前 Windows 采集机的 Docker Desktop，不新增运维主机。
- WeRSS 复用现有 MySQL 实例，但必须使用独立 `werss` 数据库、独立账号和最小权限。
- WeInsight 不读取 WeRSS 数据表，不调用 WeRSS 私有管理 API，只消费标准 RSS/Atom。
- 首期管理员在 WeRSS 添加公众号，再把 Feed URL 录入 WeInsight，不做名单自动同步。
- 公众号 RPA、手动 CLI、路由探测、进度恢复和 UI 锁依赖最终全部删除，不保留运行时回退入口。
- 公众号任务最小间隔保持 10 分钟，首次导入最近 24 小时且最多 30 篇，账号总数不超过 20。
- RSS 网络请求不得获取 `wechat_ui_lock`；微信健康异常不得阻塞 RSS；WeRSS 异常不得阻塞微信群。
- Feed URL 及每次重定向默认阻止非 HTTP/HTTPS、环回、私网、链路本地和云元数据地址；只可精确放行配置的同机 WeRSS 主机与端口，且例外不得继承到其他重定向目标；响应和解压后内容均有限额。
- 现有 `wechat_article_raw`、clean、analysis、报价和日报结果模型保持兼容。
- 实现必须遵循 TDD，每个任务先看到预期失败，再写最小实现并提交。

```python
def _is_forbidden(address: str) -> bool:
    value = ip_address(address)
    return any((value.is_private, value.is_loopback, value.is_link_local,
                value.is_multicast, value.is_reserved, value.is_unspecified))

def validate_feed_url(url: str, *, resolver=resolve_host, allowed_endpoint=None) -> str:
    parsed = urlsplit(url.strip())
    if parsed.scheme not in {"http", "https"} or not parsed.hostname:
        raise FeedUrlBlocked("feed_url_blocked")
    port = parsed.port or (443 if parsed.scheme == "https" else 80)
    exact_exception = allowed_endpoint == (parsed.hostname, port)
    if not exact_exception and any(_is_forbidden(address) for address in resolver(parsed.hostname)):
        raise FeedUrlBlocked("feed_url_blocked")
    return urlunsplit((parsed.scheme.lower(), parsed.netloc, parsed.path or "/", parsed.query, ""))
```
