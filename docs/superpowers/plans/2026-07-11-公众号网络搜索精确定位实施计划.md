# 公众号直接搜索精确定位 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** 恢复并加固公众号 RPA 的直接搜索定位：只通过顶部搜索框的公众号优先精确结果打开已配置账号，不使用“搜索网络结果”。

**Architecture:** 删除错误的网络搜索分支。保留现有 ChatWith 后的主窗口直接搜索，但在键盘输入前确认搜索编辑框拥有焦点，并将候选选择从包含匹配收紧为名称完全匹配；现有 SearchContentCellView 优先级继续高于普通 XTableCell，以优先匹配公众号分类结果而不是聊天记录。

**Tech Stack:** Python 3.12、wxauto4、pywinauto UIA、pytest。

## Global Constraints

- 只接受标准化后完全等于 account_name 的直接搜索结果；禁止模糊匹配、首项回车、force=True 和“搜索网络结果”。
- 键盘输入前必须确认顶部搜索框焦点；焦点不可确认时安全失败。
- 不新增 RSS、数据库表、前端页面、调度频率或真实采集任务。
- 真实验证仅运行 article-rpa-probe；仅在 probe status=ok、account_found=1、link_count 大于等于 1 后由管理员确认单目标 POC。
- 中文文档使用 UTF-8 无 BOM。

---

- app/rpa/wxauto_client.py：移除网络搜索，增加搜索框焦点确认和直接搜索精确匹配。
- tests/test_wxauto_client.py：直接搜索焦点、精确匹配和公众号优先级的红绿测试。
- docs/operations/微信采集管理后台受控POC执行记录.md：记录直接搜索 probe 的安全结果。
