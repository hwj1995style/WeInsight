# WeRSS 公众号只读状态与增量处理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 WeRSS 设为公众号唯一配置端，把 WeInsight 改为自动同步来源、全局每 10 分钟增量处理并只读展示状态。

**Architecture:** 新增只读 `WeRSSCatalogClient` 和事务型 `WeRSSCatalogSyncService`，由采集 Worker 按全局周期在生成公众号运行批次前同步来源；现有 RSS 入库、清洗、分析与报价任务继续以幂等状态表推进。管理后台通过独立状态查询服务展示来源、采集和处理积压，不再暴露公众号写接口。

**Tech Stack:** Python 3.12、FastAPI、Jinja2、SQLAlchemy Core、MySQL 8、httpx、APScheduler、pytest。

## Global Constraints

- WeRSS 只允许通过 `http://127.0.0.1:8001/api/v1/wx/mps` 的 GET 请求读取，禁止跨库和任何 WeRSS 写请求。
- AK/SK 只从 `WEINSIGHT_WERSS_ACCESS_KEY`、`WEINSIGHT_WERSS_SECRET_KEY` 环境变量读取，禁止落入 YAML 明文、数据库、日志、HTML和测试输出。
- 公众号全局周期默认 `10` 分钟，配置值不得小于 `10`。
- “一箱蛋”必须在服务端同步层排除，不得创建新的采集、清洗、分析或报价任务。
- 旧公众号 RPA 保持删除状态，不得新增旧命令、调度、探针或 UI 锁入口。
- 上游停用、缺失或 API 故障不得删除历史文章和任务引用。
- 所有新增 SQL `insert ... select` 输出字段逐字段添加业务注释；本计划不新增 Doris 物理临时表。
- 实现遵循 TDD；每个任务先运行指定失败测试，再实现最小代码并单独提交。

## File Structure

- `app/integrations/werss_catalog.py`：固定端点、只读鉴权、分页和响应契约校验。
- `app/services/werss_catalog_sync_service.py`：排除、旧记录绑定、完整清单同步语义。
- `app/storage/werss_catalog_sync_repo.py`：单事务来源同步和数据库互斥。
- `app/services/article_source_status_service.py`：公众号只读状态领域模型和分页。
- `app/storage/article_source_status_repo.py`：来源、文章、采集日志和任务积压聚合查询。
- `app/workers/article_global_cycle.py`：按全局周期执行清单同步并为全部 active 来源维护系统采集任务。
- `app/web/routes/sources.py`：保留群管理，公众号只保留 GET 状态页。
- `app/web/templates/sources/articles.html`：只读公众号状态表。
- `sql/migrations/20260713_001_add_werss_catalog_state.sql`：来源映射字段和索引。
