# 微信信息采集分析系统第二阶段 RPA POC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立微信群 RPA POC 的可测试闭环：微信桌面健康探测、群消息采集服务、raw/cursor/log 入库结构、fake RPA 验证、真实 RPA 适配器兼容性检查和清晰失败边界。

**Architecture:** 延续第一阶段双链路隔离。第二阶段只推进微信群链路，公众号/订阅号链路不改动；真实微信 UI 访问通过适配器封装，业务采集服务只依赖 `WechatGroupRpaClient` 协议和 repository 协议。微信 PC 4.1.8.107 必须先通过 RPA 适配器兼容性检查，不能把底层窗口句柄错误暴露给业务采集命令。

**Tech Stack:** Python 3.11, pytest, SQLAlchemy/PyMySQL, PowerShell process probing, optional wxauto4/wxautox4 for WeChat 4.x, pywinauto for process/window diagnostics.

用户确认微信不能降级后，优先尝试恢复 UIA 控件树。


尝试结果：

| 尝试项 | 参数或设置 | 结果 |
| --- | --- | --- |
| Qt 软件渲染 | `QT_ANGLE_PLATFORM=software`、`QT_OPENGL=software`、`QT_QUICK_BACKEND=software`、`QSG_RHI_BACKEND=software` | 登录窗口可见 `mmui::LoginWindow`，进入主界面后仍回到 `Qt51514QWindowIcon` |
| Qt 软件渲染 + DPI 环境变量 | 追加 `QT_ENABLE_HIGHDPI_SCALING=0`、`QT_AUTO_SCREEN_SCALE_FACTOR=0`、`QT_SCALE_FACTOR=1` | 主界面仍为 `Qt51514QWindowIcon` |
| WARP/软件渲染偏好 | `QT_OPENGL=angle`、`QT_ANGLE_PLATFORM=warp`、`QSG_RHI_PREFER_SOFTWARE_RENDERER=1`、`QSG_RENDER_LOOP=basic`、启动参数 `--disable-gpu --disable-gpu-compositing` | 登录窗口可见 `mmui::LoginWindow`，短暂出现 `mmui::MainWindow` 后主界面仍回到 `Qt51514QWindowIcon` |
| Windows 高 DPI 兼容层 System | HKCU AppCompatFlags `~ DPIUNAWARE` | 主界面仍为 `Qt51514QWindowIcon` |
| Windows 高 DPI 兼容层 System Enhanced | HKCU AppCompatFlags `~ GDIDPISCALING DPIUNAWARE` | 主界面仍为 `Qt51514QWindowIcon` |

回退：

```text
已删除 D:\App\Wechat\Weixin\Weixin.exe 的 HKCU AppCompatFlags\Layers 兼容层配置
已用默认方式重启微信
当前窗口：mmui::LoginWindow
```

阶段结论：

当前主机和微信 PC 4.1.10.31 组合下，进程级 Qt/ANGLE 环境变量和 Windows 高 DPI 兼容层均未稳定恢复主界面 UIA 控件树。下一步建议转入剪贴板探针；如果剪贴板不可用，再进入 OCR 键鼠适配器。
