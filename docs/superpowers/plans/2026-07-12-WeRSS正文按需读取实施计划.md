# WeRSS 正文按需读取实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 WeInsight 的文章解析与分析任务优先按需读取本机 WeRSS 正文、必要时回退公众号网页，同时只持久化正文来源、长度和哈希等安全元数据。

**Architecture:** Feed 适配层提取受控 WeRSS locator 并写入 raw 元数据；统一 `ArticleContentProvider` 在解析和分析阶段按需返回仅存于内存的规范化正文。`FallbackArticleContentProvider` 执行 WeRSS 优先/网页回退策略，repo 只保存 `content_source`、`content_hash`、`content_length`，分析任务需要正文时再次通过同一 provider 临时读取，避免跨任务持久化正文。

**Tech Stack:** Python 3.12、httpx、feedparser、SQLAlchemy、MySQL 8.4、pytest、Docker Desktop WeRSS。

## Global Constraints

- WeRSS 正文端点只允许配置中的 `127.0.0.1:8001`，沿用地址固定、逐跳重定向校验和 5 MiB 响应上限。
- WeInsight 数据库、日志、异常摘要和运行目录不得保存文章正文或原始 HTML。
- WeRSS 管理密码、Cookie、令牌不得写入数据库、任务快照、日志或测试输出。
- 安全策略错误不得静默回退；只有 locator 缺失、正文不存在/为空及可恢复网络或解析错误允许回退。
- 公众号 RSS 与微信群 RPA 保持隔离，正文读取不得获取 `wechat_ui_lock`。
- 所有数据库迁移必须兼容 MySQL 8.4 且可重复执行。
- 使用官方固定摘要，不构建内部镜像。已接受正文进入本机 Docker 日志的风险；日志仅保留在本机 Docker 日志，不接入外部日志，不复制到工单，不纳入备份，轮转限制为 2 × 2 MB，并持续关注上游安全开关。
- 采集与下游分开：9 个公众号全部启用采集，但仅湖南三尖农牧公司进入 clean/analyze，其余 8 个只采集。必须先有下游白名单，禁止手工删任务冒充隔离。

## 文件结构

- `app/rss/models.py`：承载 Feed 条目的受控正文 locator。
- `app/rss/feed_client.py`：从标准 Feed/WeRSS 扩展字段提取 locator，不发起正文请求。
- `app/rss/article_mapper.py`：把 locator 映射到 raw 记录。
- `app/content/article_content.py`：统一正文结果、错误分类和 provider 协议。
- `app/content/werss_provider.py`：受控读取 WeRSS、净化 HTML、生成规范化文本。
- `app/content/fallback_provider.py`：实现允许/禁止回退矩阵。
- `app/pipelines/article_parse_service.py`：使用 source-aware provider 并写安全元数据。
- `app/pipelines/article_transient_extractor.py`：分析阶段再次按需读取正文，不持久化正文。
- `app/storage/article_raw_repo.py`、`app/storage/article_parse_repo.py`：读写 locator 和安全正文元数据。
- `sql/init.sql`、`sql/migrations/20260712_001_add_article_content_metadata.sql`：幂等增加元数据列。
- `app/core/config.py`、`config/config.dev.yaml`、`config/config.prod.example.yaml`：正文端点、超时、大小和影子模式配置。
- `app/main.py`、`app/workers/pipeline_runtime_factory.py`：组装共享 provider。
- `docs/operations/公众号RSS采集运行手册.md`：部署、巡检、回退和 POC 指标。
