# 微信信息采集分析系统第三阶段运维化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将已跑通的微信群采集、清洗、分析、日报链路推进到可试运行、可巡检、可补偿、可恢复的运维化状态。

**Architecture:** 第三阶段继续只推进微信群链路，不启动公众号/订阅号真实采集。所有新增命令默认只读或显式补偿，不输出 raw/clean 消息正文；真实微信 UI 操作仍只允许发生在明确授权的采集入口。开发环境继续手动执行，不注册 Windows 计划任务。

**Tech Stack:** Python 3.12, pytest, SQLAlchemy/PyMySQL, MySQL 8.4, PowerShell, Markdown runbook.

4. 开发环境不注册 Windows 计划任务。
5. 公众号/订阅号链路继续保持隔离，不因群链路任务变更被修改。
