# WeInsight 管理后台阶段五部署与受控上线 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Web、采集 Worker、后处理 Worker 安全部署到 Windows，完成 HTTPS、MySQL 最小权限、旧 scheduler 切换、浏览器 E2E、Fake RPA 验收和单目标真实 POC。

**Architecture:** 三个进程使用独立启动脚本和日志目录，由 Windows Task Scheduler 在交互式用户登录时启动。Web 绑定指定内网 IP 并启用 TLS；三个 MySQL 角色按职责隔离；真实采集只在阶段闸门通过后启用。

**Tech Stack:** Windows 10/11、PowerShell 5.1+、Windows Task Scheduler、Windows Defender Firewall、Uvicorn TLS、MySQL roles、Playwright、pytest、现有微信 PC 4.1.8.107。

## Global Constraints

- 必须先完成阶段四计划并通过完整测试。
- 生产 Web 只绑定明确的内网 IP，禁止绑定公网地址。
- 生产必须使用 HTTPS；证书和私钥路径通过配置提供，私钥不进入仓库。
- 采集 Worker 必须使用 ScheduledTasks 模块的 `-LogonType Interactive` 运行，不能使用 Session 0 服务账号。
- Web、采集 Worker、后处理 Worker 使用独立日志目录和独立 MySQL 账号。
- Web MySQL 账号不得 SELECT 群 raw；采集 Worker 不得读取管理员密码或 Session；后处理 Worker 不得写 UI 锁。
- 启用托管采集前必须停止并注销旧长期 group scheduler。
- Fake RPA 验收通过前禁止真实采集；单群单公众号 POC 通过前禁止自动调度和扩容。
- 真实 POC 全程人工值守，微信固定 4.1.8.107，关闭自动更新、锁屏和休眠。
- 回滚必须能独立停止 article、停止全部托管采集、保留 Web 只读查询和恢复旧手动 CLI。

- [ ] **Step 2: 编写 role SQL**

创建三个 role，不创建用户、不写密码：

```sql
CREATE ROLE IF NOT EXISTS 'weinsight_web_role';
CREATE ROLE IF NOT EXISTS 'weinsight_collector_role';
CREATE ROLE IF NOT EXISTS 'weinsight_pipeline_role';
```

Web role：认证/Session/配置/job/target/report request 需要精确 CRUD；run/event/heartbeat/health/collect log/clean/analysis/egg price/daily report 只读。Collector role：source/job/target 只读；run/target run/event/heartbeat/health/raw/cursor/progress/collect log/process task/UI lock 按实际采集路径读写。Pipeline role：raw/process task/clean/analysis/egg price/daily report/report request/event/heartbeat 按后处理路径读写；不授予管理员表和 UI lock。
