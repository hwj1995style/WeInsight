# 四账号验收与二十账号扩容判断

> **公众号 article-RPA 历史记录（已退役）**：本文中的 `article-rpa-probe`、`collect-article-once`、`run-article-scheduler` 等命令已移除，**禁止执行**，不得作为现行操作指引。公众号现行采集路径仅为 **WeRSS + WeInsight RSS**；请使用 `docs/operations/公众号RSS采集运行手册.md`。

本文档用于第九阶段 Task 70。目标是在 Task 69 的小规模交错 POC 完成后，结合 2026-07-08 实际完成的 4 个公众号/订阅号 controlled POC，判断是否继续观察、是否回滚到单账号模式、是否只保留群链路，以及是否允许进入最多 20 个账号的后续扩容设计。

## 1. 输入材料

验收前必须准备：

```text
Task 69：小规模交错 POC 运行记录
docs/operations/三账号小规模交错POC运行记录.md

```

验收只基于上述记录和指标，不临时新增账号，不临时启动无人值守，不临时注册 Windows 计划任务。

## 2. 判断范围

本任务只判断：

```text
4 个账号是否稳定
是否继续观察
是否回滚到单账号模式
是否只保留群链路
是否允许进入最多 20 个扩容设计
```

本任务不允许：

```text
直接启用最多 20 个账号
直接注册 Windows 计划任务
直接启用无人值守
直接调用外部模型
绕过 wechat_ui_lock 操作微信 UI
输出聊天正文、文章全文、HTML 或具体文章链接
```

4 个小规模账号稳定后再考虑最多 20 个。最多 20 个只做后续评估，不在第九阶段直接启用。

## 4. 决策规则

Go：4 个账号和核心群交错稳定，日报质量达标，核心群未超阈值等待。

Go 条件：

```text
Task 69 结论为 Go
4 个账号均按每小时执行 1 次稳定运行
4 个账号均只采集当天发布数据
核心群等待未超过阈值
群链路优先保持有效
article 链路可中断且可恢复
日报质量平均分不低于 4.0
没有任一账号连续失败 3 次
任一账号失败都可单独关闭
AI 继续 dry-run 且 model_called=0
```

Watch：存在可定位失败，可单账号关闭并继续观察。

Watch 条件：

```text
存在可定位失败
失败账号可单独关闭
日报质量平均分低于 4.0 但不低于 3.0
群链路未受明显影响
核心群等待未超过阈值
需要继续人工复盘
```

No-Go：任一账号连续失败 3 次，或核心群等待超过阈值。

No-Go 条件：

```text
任一账号连续失败 3 次
核心群等待超过阈值
article 链路不可中断
article 链路长时间占用微信 UI
UI 锁超时持续出现
日报质量平均分低于 3.0
AI 不再是 dry-run
model_called 不等于 0
用户可见输出出现敏感内容
```

扩容：最多 20 个只进入后续设计，不在第九阶段直接启用。

## 5. 最多 20 个扩容设计准入

允许进入最多 20 个扩容设计：

```text
关闭 article 链路后群链路仍可独立运行

```

不得进入最多 20 个扩容设计：

```text
Task 69 为 Watch 或 No-Go
Task 70 为 Watch 或 No-Go
任一账号连续失败 3 次
核心群等待超过阈值
回滚路径未演练
失败原因不可定位
用户可见输出存在敏感内容
```

最多 20 个设计阶段必须另起设计文档，重新评估调度间隔、UI 锁占用、账号分批、失败隔离、日报质量和回滚预案。

## 6. 回滚判断

回滚到单账号模式：

```text
4 个账号中存在不稳定账号
任一账号连续失败 3 次
日报质量低于扩容阈值
小规模账号交错小窗口出现可恢复但需继续观察的问题
```

只保留群链路：

```text
核心群等待超过阈值
article 链路长时间占用微信 UI
UI 锁超时持续出现
关闭 article 链路后仍需观察核心群稳定性
```

继续观察：

```text
存在少量失败但原因可定位
失败账号可单独关闭
日报质量需要人工复盘
指标波动可解释但样本时间不足
```
