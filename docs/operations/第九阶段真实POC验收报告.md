# 第九阶段真实POC验收报告

本文档用于第九阶段 Task 67。目标是在 Task 64、Task 65 和 Task 66 完成后，基于真实执行记录、指标、日报质量和回滚演练，给出是否允许从 1 个扩到 3 个的 Go / Watch / No-Go 结论。

## 1. 输入材料

验收前必须准备：

```text
Task 61：真实 POC 验收与扩容判断
Task 64：单公众号/订阅号真实 POC 执行记录
Task 65：单核心群真实轮询 POC 执行记录
Task 66：双链路交错真实小窗口验证记录
Task 53：日报质量人工评估表
trial-monitor-report 输出
group-runtime-metrics 输出
article-runtime-metrics 输出
```

验收只基于上述记录和指标，不临时扩大账号，不临时启动无人值守，不临时注册 Windows 计划任务。

## 2. 验收范围

本报告只判断：

```text
1 个实际授权公众号/订阅号是否通过真实执行
1 个实际授权核心群是否通过真实轮询
1 到 2 小时双链路交错小窗口是否通过
是否允许从 1 个扩到 3 个
是否需要 Watch 继续观察
是否需要回滚到单账号模式或只保留群链路
```

本报告不允许：

```text
直接启用 3 个账号
直接启用最多 20 个账号
注册 Windows 计划任务
启用无人值守
启用外部模型调用
将 AI dry-run 结果作为自动决策
```

最多 20 个只做后续评估，不在 Task 67 直接启用。

## 3. 决策表

| 项目 | 输入记录 | 关键证据 | 结论 |
| --- | --- | --- | --- |
| 单公众号/订阅号真实执行 | Task 64 | 只采集当天发布数据、每轮最多 3 篇、采集后释放 UI、解析不占用 UI | Go / Watch / No-Go |
| 单核心群真实执行 | Task 65 | 1 个实际授权核心群、单轮轮询、去重、截图只用于内部排查 | Go / Watch / No-Go |

| 回滚演练 | Task 64 / Task 65 / Task 66 | 回滚到单账号模式、只保留群链路、关闭 AI | Go / Watch / No-Go |

## 4. Go / Watch / No-Go

Go 条件：

```text
单公众号/订阅号真实执行为 Go
单核心群真实执行为 Go
双链路交错小窗口为 Go
日报质量平均分不低于 4.0
核心群等待未超过阈值
没有任一账号连续失败 3 次
wechat_ui_lock 未出现长时间占用
AI 继续 dry-run 且 model_called=0
回滚演练可执行
```

Go 后才允许进入 Task 68，配置 3 个实际授权公众号/订阅号。

Watch 条件：

```text
存在可定位失败
失败可手动恢复
日报质量平均分低于 4.0 但不低于 3.0
article 链路失败可回滚到单账号模式
群链路未受明显影响
需要补充截图、日志或指标证据
```

Watch 时不得进入 Task 68。

No-Go 条件：

```text
任一账号连续失败 3 次
核心群等待超过阈值
article 链路不可中断
article 链路长时间占用微信 UI
UI 锁超时持续出现
日报质量平均分低于 3.0
AI 不再是 dry-run
model_called 不等于 0
用户可见输出出现敏感内容
```

No-Go 时必须停止扩容。

## 5. 扩容判断

允许从 1 个扩到 3 个：

```text
Task 64 结论为 Go
Task 65 结论为 Go
Task 66 结论为 Go
日报质量平均分不低于 4.0
连续 1 到 2 小时无核心群超阈值等待
没有任一账号连续失败 3 次
AI 继续 dry-run 且 model_called=0
回滚路径已演练
```

不允许从 1 个扩到 3 个：

```text
任一输入记录为 No-Go
任一关键证据缺失
失败原因不可定位
回滚路径不可执行
AI dry-run 未验证 model_called=0
用户可见输出存在敏感内容
```

最多 20 个只做后续评估：

```text
Task 67 不直接启用最多 20 个
Task 68 只配置 3 个实际授权公众号/订阅号
3 个账号稳定后，才允许进入最多 20 个扩容判断
```

## 6. 回滚判断

回滚到单账号模式：

```text
article 链路失败集中在某一个账号
任一账号连续失败 3 次
双链路交错出现可恢复但需继续观察的问题
日报质量低于扩容阈值
```

只保留群链路：

```text
核心群等待超过阈值
article 链路长时间占用微信 UI
UI 锁超时持续出现
关闭 article 链路后仍需观察群链路稳定性
```

关闭 AI：

```text
AI 不再是 dry-run
model_called 不等于 0
AI 输出被用于自动改写链路状态
```
