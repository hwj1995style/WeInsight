# 日报质量人工评估表

本文档用于第七阶段 Task 53：试运行日报质量人工复盘与指标阈值调整。目标是在无 AI 小规模试运行期间，对群日报、文章日报和汇总日报做统一人工评分，形成 Go / No-Go 判断依据。

## 1. 背景与现状

当前系统已经具备三类日报：

```text
群日报
文章日报
汇总日报
```

三类日报分别来自独立链路。质量复盘只读取已经生成的日报 Markdown，不触发采集、清洗、分析或调度，不占用微信 PC UI，也不获取 `wechat_ui_lock`。

## 2. 目标与非目标

目标：

```text
统一人工评分维度
记录日报准确性、噪声和可读性
给出阈值调整和暂停建议
支撑小规模试运行 Go / No-Go 判断
```

非目标：

```text
不自动修改群链路或 article 链路任务状态
不自动调整生产配置
不摘录正文、HTML 或具体文章链接
不作为 AI 分析输入
```

## 3. 影响范围

评估范围：

```text
群日报：供需判断、关键词噪声、联系方式脱敏、可读性
文章日报：摘要可用性、标签准确性、HTML 表格提取质量、图片 OCR 表格质量、可读性
汇总日报：总览可用性、双链路信息平衡、行动建议可执行性
```

不影响范围：

```text
微信群采集任务
公众号/订阅号采集任务
清洗和分析任务
日报生成任务
微信 UI 锁
```

## 4. 数据模型和权限模型

配置模板位于：

```text
config/report_quality_review.yaml
```

人工评估记录建议先保存在试运行台账或独立表格中，不回写 group/article 任务表。评分采用 1 到 5 分：

```text
5 分：可直接用于试运行判断
4 分：基本可用，有轻微问题
3 分：需要关注，继续人工复盘
2 分：质量较差，需要暂停扩容或修正规则
1 分：不可用，需要回滚到人工复核
```

## 5. 评估表

### 群日报

| 维度 | 评分 | 复盘说明 | 调整建议 |
| --- | --- | --- | --- |
| 准确性 |  | 供需判断是否符合人工抽样 | 调整供需关键词或排除词 |
| 噪声 |  | 是否有大量无关关键词命中 | 收紧关键词规则 |
| 联系方式脱敏 |  | 是否避免暴露联系方式细节 | 补充脱敏规则 |
| 可读性 |  | 是否便于快速判断当天状态 | 调整模板段落顺序 |

### 文章日报

| 维度 | 评分 | 复盘说明 | 调整建议 |
| --- | --- | --- | --- |
| 摘要可用性 |  | 是否能表达文章核心信息 | 调整摘要生成规则 |
| 标签准确性 |  | 标签是否贴合文章主题 | 调整标签词表 |
| HTML 表格提取质量 |  | 表格字段是否结构清晰 | 优化表格解析规则 |
| 图片 OCR 表格质量 |  | 图片表格识别是否可用于判断 | 增加人工复核或降低权重 |
| 可读性 |  | 是否避免堆砌无关结构化字段 | 调整模板展示顺序 |

### 汇总日报

| 维度 | 评分 | 复盘说明 | 调整建议 |
| --- | --- | --- | --- |
| 总览可用性 |  | 是否能快速说明当天整体状态 | 收紧总览模板 |
| 双链路信息平衡 |  | 是否同时覆盖群链路和 article 链路 | 调整两条链路展示比例 |
| 行动建议可执行性 |  | 是否能支撑后续操作判断 | 固定可执行建议格式 |

## 6. 阈值调整规则

按照 `config/report_quality_review.yaml` 的阈值执行人工判断：

```text
平均分不低于 4.0：继续当前试运行节奏
平均分低于 4.0 但不低于 3.0：进入观察，继续人工复盘
平均分低于 3.0：暂停扩容并修正规则
关键维度低于 2 分：暂停对应链路扩容
```

建议处理：

```text
群日报噪声偏高：优先调整群分析关键词和排除词
文章日报表格质量偏低：保留文章链路，但相关结论必须人工确认
图片 OCR 表格质量偏低：降低 OCR 表格在日报中的权重
汇总日报行动建议偏弱：只保留已验证的摘要信号
article 链路质量连续不达标：暂停 article 链路扩容，只保留群链路
```

## 7. 后端实现方案

Task 53 只落地人工评估文档和 YAML 阈值模板。后端暂不新增自动评分服务，也不新增数据库写入。后续如需自动化，可新增独立质量评估表，且必须保持只读日报输入、独立入库、失败不回写 group/article 链路状态。

## 8. 前端影响

当前无前端改动。人工复盘阶段以 Markdown 文档和 YAML 模板为准。

## 10. 风险与分阶段落地建议

风险：

```text
人工评分主观性较强
小样本评估可能不足以代表长期效果
表格和 OCR 质量容易受文章格式影响
```

分阶段建议：

```text
第一阶段：每天人工评分一次，至少连续 3 天
第二阶段：对低于阈值的维度只调规则，不扩大账号数量
第三阶段：连续达标后再进入 AI 灰度设计
```
