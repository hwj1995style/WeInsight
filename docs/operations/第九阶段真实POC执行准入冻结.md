# 第九阶段真实POC执行准入冻结

本文档用于第九阶段 Task 62。目标是在开始真实账户受控 POC 执行与 3 账号小规模扩容前，冻结执行边界，避免真实执行过程中临时扩大范围、混入无人值守或提前打开 AI。

## 1. 阶段目标

第九阶段只做真实账户受控 POC 执行与 3 账号小规模扩容。

允许：

```text
1 个实际授权公众号/订阅号真实执行
1 个实际授权核心群真实轮询
1 到 2 小时双链路交错真实小窗口验证
验收通过后扩到 3 个实际授权公众号/订阅号
3 个账号稳定后再评估最多 20 个
```

不允许：

```text
不得临时扩大账号
不得跳过 1 个账号直接进入 3 个账号
不得跳过 3 个账号直接进入最多 20 个
不得注册 Windows 计划任务
不得启用无人值守
不得启用后台常驻 article 调度
不得将 AI dry-run 结果作为自动决策
```

## 2. 准入冻结

准入冻结条件：

```text
微信 PC 4.1.8.107
微信自动更新已关闭
开发 MySQL 可连接
config/config.dev.yaml 可加载
真实 POC 前置复核清单结论为 Go
真实 POC 验收与扩容判断模板已准备
AI 继续 dry-run
model_called=0
不注册 Windows 计划任务
不启用无人值守
```

账号冻结：

```text
第一轮只允许 1 个实际授权公众号/订阅号
第一轮只允许 1 个实际授权核心群
公众号/订阅号只采集当天发布数据
公众号/订阅号每轮最多 3 篇
公众号/订阅号每小时最多执行 1 次
正文只运行时临时读取
不长期保存文章正文
```

UI 冻结：

```text
所有微信 PC UI 操作必须获取 wechat_ui_lock
同一时刻只允许一个链路持有 wechat_ui_lock
群链路优先
article 链路低峰执行
article 链路可中断
article 链路不得长时间占用微信 UI
每次 UI 占用必须有超时
超时后释放锁并记录失败
```

## 3. 执行顺序

第九阶段按以下顺序执行：

```text
Task 63：真实 POC 环境核验
Task 64：单公众号/订阅号真实 POC 执行
Task 65：单核心群真实轮询 POC 执行
Task 66：双链路交错真实小窗口验证
Task 67：真实 POC 验收报告落地
Task 68：3 个公众号/订阅号扩容配置
Task 69：3 账号小规模交错 POC
Task 70：3 账号验收与最多 20 个扩容判断
```

Task 68 只能在 Task 67 结论为 Go 后执行。

Task 69 只能在 Task 68 配置核验通过后执行。

Task 70 只做扩容判断，不直接启用最多 20 个账号。

## 4. 暂停条件

出现以下任一情况，立即暂停 article 链路：

```text
任一账号连续失败 3 次
核心群等待超过阈值
wechat_ui_lock 超时持续出现
article 链路不可中断
article 链路长时间占用微信 UI
微信掉线
微信窗口卡死
锁屏导致 UI 不可操作
用户可见输出出现聊天正文、文章全文、HTML 或具体文章链接
AI 不再是 dry-run
model_called 不等于 0
```

暂停后先保留现场：

```text
保留日志
保留截图
记录失败任务
记录 UI 锁持有和释放时间
记录 group/article 指标
```

## 5. 回滚规则

回滚到单账号模式：

```text
3 个账号配置后任一账号连续失败 3 次
某个账号失败可定位但需要继续观察
日报质量低于扩容阈值
双链路交错小窗口出现可恢复问题
```

只保留群链路：

```text
核心群等待超过阈值
article 链路长时间占用微信 UI
UI 锁异常影响群链路
关闭 article 链路后仍需继续观察核心群
```

关闭 AI：

```text
AI 不再是 dry-run
model_called 不等于 0
AI 输出被用于自动改写链路状态
```

## 6. Go / Watch / No-Go

Go：

```text
环境核验为 Go
单公众号/订阅号执行为 Go
单核心群执行为 Go
双链路交错小窗口为 Go
日报质量达标
AI 继续 dry-run 且 model_called=0
```

Watch：

```text
存在可定位失败
失败可手动恢复
群链路未受明显影响
需要延长观察但无需立即中止
```

No-Go：

```text
任一账号连续失败 3 次
核心群等待超过阈值
article 链路不可中断
UI 锁超时持续出现
用户可见输出出现敏感内容
AI 不再是 dry-run
```
