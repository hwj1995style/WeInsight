# 生产配置和回滚预案

本文档用于第七阶段 Task 56：生产配置和回滚预案复核。目标是在小规模试运行进入生产前，明确配置检查、权限边界、监控入口和回滚步骤。

## 1. 适用范围

适用规模：

```text
核心群不超过5个
公众号/订阅号不超过20个
微信 PC 版本 4.1.8.107
单机共用微信 UI 串行交错执行
```

默认策略：

```text
群链路优先
article 链路可关闭
AI 默认关闭
巡检、日报和 AI dry-run 不占用微信 UI
```

## 2. 生产配置文件

生产配置模板使用现有文件：

```text
config/config.prod.example.yaml
```

上线前复制为本机私有配置：

```powershell
Copy-Item config\config.prod.example.yaml config\config.prod.yaml
$env:WEINSIGHT_MYSQL_PASSWORD='<生产库密码>'
python -m app.main check-config --config config/config.prod.yaml
```

检查重点：

```text
app.env=prod
wechat.pc_version=4.1.8.107
pipelines.group.enabled=true
pipelines.group.core_group_limit=5
pipelines.article.enabled=false 或人工确认后开启
pipelines.article.account_limit=20
pipelines.article.collect_today_only=true
pipelines.article.dedup_key=article_hash
pipelines.ui_resource.mode=exclusive
pipelines.ui_resource.group_priority=true
mysql.password 使用 WEINSIGHT_MYSQL_PASSWORD
```

## 3. MySQL 数据库和账号权限

生产库不得使用开发 Docker MySQL。生产配置应指向独立 MySQL：

```text
host: prod-mysql.internal
port: 3306
database: weinsight_prod
username: weinsight_prod
password: ${WEINSIGHT_MYSQL_PASSWORD}
```

权限建议：

```text
采集进程账号：只访问本项目生产库
巡检和日报命令：只读日报、指标和任务状态
AI dry-run：只读摘要、结构化特征、脱敏字段
禁止把生产密码写入 YAML、日志、截图或命令输出
```

初始化和升级：

```powershell
python -m app.main print-init-sql --config config/config.prod.yaml
```

正式执行建表或迁移前，先在备份库或演练库验证 SQL。

## 4. 日志目录

生产配置使用以下目录：

```text
runtime/logs
runtime/screenshots
runtime/reports
```

检查项：

```text
目录存在且采集账号可写
日志按日期保留
截图只用于失败排查
日报导出目录不对外公开
不把具体文章链接、聊天正文或完整联系方式写入日志
```

## 5. 启动前检查

执行：

```powershell
python -m app.main check-config --config config/config.prod.yaml
python -m app.main wechat-health --config config/config.prod.yaml
python -m app.main group-runtime-summary --config config/config.prod.yaml --limit 5
python -m app.main trial-monitor-report --config config/config.prod.yaml --hours 24
python -m app.main ai-analysis-sample --source summary_daily_report --date 2026-07-07 --dry-run
```

通过条件：

```text
配置能解析
微信健康检查通过
UI 锁为空或可恢复
群链路指标可读
article 链路默认关闭或人工确认开启
AI dry-run 输出 model_called=0
```

## 6. 异常处理

### 微信掉线

处理：

```text
暂停真实采集命令
人工重新登录微信
执行 wechat-health
确认 UI 锁是否释放
优先恢复群链路
```

### 锁屏

处理：

```text
恢复桌面会话
确认微信窗口可见
执行 wechat-health
检查 trial-monitor-report
只在健康检查通过后恢复调度
```

### 窗口卡死

处理：

```text
停止当前采集调度
记录截图和日志路径
人工关闭并重启微信
执行 wechat-health
等待 stale lock 自动恢复或人工处理锁记录
先运行群链路巡检，再评估 article 链路
```

## 7. 回滚路径

### 关闭 article 链路

适用条件：

```text
核心群等待超过阈值
任一账号连续失败 3 次
article 链路积压持续增加
公众号/订阅号采集影响群链路
```

操作：

```powershell
python -m app.main article-account-list --config config/config.prod.yaml
python -m app.main article-account-disable --config config/config.prod.yaml --account-name "授权公众号名称"
python -m app.main article-runtime-metrics --config config/config.prod.yaml --hours 24
```

配置层回滚：

```text
pipelines.article.enabled=false
```

### 关闭 AI

AI 当前只允许 dry-run。出现以下情况必须关闭 AI：

```text
输出不符合输入白名单
模型调用不是 dry-run
AI 输出被误用于改写日报
AI 失败影响 group/article 任务状态
```

操作：

```text
config/ai_analysis.yaml 中保持 ai.enabled=false
config/ai_analysis.yaml 中保持 ai.dry_run=true
```

验证：

```powershell
python -m app.main ai-analysis-sample --source summary_daily_report --date 2026-07-07 --dry-run
```

预期输出包含：

```text
model_called=0
```

### 只保留群链路

适用条件：

```text
微信 UI 不稳定
article 链路反复打断
AI 或日报质量未达标
需要优先保障核心群
```

保留命令：

```powershell
python -m app.main group-runtime-summary --config config/config.prod.yaml --limit 5
python -m app.main group-runtime-metrics --config config/config.prod.yaml --hours 24
python -m app.main run-group-scheduler --config config/config.prod.yaml --once
```

停止或不执行：

```text
run-article-scheduler
collect-article-once
ai-analysis-sample 之外的 AI 能力
```

## 8. 恢复到手动单账号模式

当 article 链路需要重新验证时，恢复到手动单账号模式：

```powershell
python -m app.main article-account-upsert --config config/config.prod.yaml --account-name "授权公众号名称" --account-type subscription --priority 1 --poll-interval-minutes 60 --max-articles-per-round 3
python -m app.main collect-article-once --config config/config.prod.yaml --account-name "授权公众号名称" --max-articles-per-round 3
python -m app.main parse-article-once --config config/config.prod.yaml --limit 5
python -m app.main article-runtime-metrics --config config/config.prod.yaml --hours 24
```

通过条件：

```text
单账号成功采集当天发布数据
去重生效
解析不占用微信 UI
核心群等待未超过阈值
失败任务可查看和限量重试
```

## 9. 生产 Go / No-Go

Go 条件：

```text
生产配置检查通过
MySQL 账号权限确认
微信健康检查通过
核心群不超过5个
公众号/订阅号不超过20个
article 链路可单独关闭
AI 保持关闭或 dry-run
回滚到只保留群链路的步骤已演练
```

No-Go 条件：

```text
生产配置仍指向开发库
WEINSIGHT_MYSQL_PASSWORD 未配置
微信掉线、锁屏或窗口卡死无法恢复
UI 锁无法释放
关闭 article 链路后群链路仍异常
关闭 AI 后仍存在模型调用
CLI 或日志输出出现正文、完整联系方式或具体文章链接
```
