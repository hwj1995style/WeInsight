# 三公众号订阅号扩容配置记录

本文档用于第九阶段 Task 68。目标是在 Task 67 真实 POC 验收报告给出 Go 结论后，将 article 链路从 1 个实际授权公众号/订阅号扩到 3 个实际授权公众号/订阅号，并记录配置、核验和单账号关闭路径。

## 1. 执行边界

本任务只允许：

```text
3 个实际授权公众号/订阅号
人工值守
手动命令触发
每个账号每小时最多执行 1 次
每个账号每轮最多 3 篇
只采集当天发布数据
不注册 Windows 计划任务
不启用无人值守
不启用后台常驻 article 调度
AI 继续 dry-run
model_called=0
```

本任务不允许：

```text
Task 67 非 Go 时配置 3 个账号
一次性配置超过 3 个账号
直接进入最多 20 个账号
启动无人值守
启动长期后台调度
绕过 wechat_ui_lock 操作微信 UI
输出文章全文、HTML 或具体文章链接
```

最多 20 个只做后续评估，不在 Task 68 直接启用。

## 2. 前置条件

执行前必须完成：

```text
docs/operations/第九阶段真实POC验收报告.md
```

Task 67 必须为 Go 结论，并确认：

```text
Task 64 单公众号/订阅号真实执行为 Go
Task 65 单核心群真实执行为 Go
Task 66 双链路交错小窗口为 Go
日报质量平均分不低于 4.0
核心群等待未超过阈值
没有任一账号连续失败 3 次
AI 继续 dry-run 且 model_called=0
回滚路径已演练
```

如果 Task 67 为 Watch 或 No-Go，不进入本任务。

## 3. 账号配置命令

在项目根目录执行前，先设置开发库密码：

```powershell
$env:WEINSIGHT_MYSQL_PASSWORD='weinsight_dev'
```

配置 3 个实际授权公众号/订阅号：

```powershell
python -m app.main article-account-upsert --config config/config.dev.yaml --account-name "授权公众号名称A" --account-type subscription --priority 1 --poll-interval-minutes 60 --max-articles-per-round 3
python -m app.main article-account-upsert --config config/config.dev.yaml --account-name "授权公众号名称B" --account-type subscription --priority 2 --poll-interval-minutes 60 --max-articles-per-round 3
python -m app.main article-account-upsert --config config/config.dev.yaml --account-name "授权公众号名称C" --account-type subscription --priority 3 --poll-interval-minutes 60 --max-articles-per-round 3
```

配置后检查：

```powershell
python -m app.main article-account-list --config config/config.dev.yaml
```

记录：

```text
账号 A：
账号 B：
账号 C：
是否均为实际授权公众号/订阅号：
是否只配置 3 个实际授权公众号/订阅号：
是否每小时最多执行 1 次：
是否每轮最多 3 篇：
是否只采集当天发布数据：
```

## 4. 单账号关闭路径

任一账号失败都可单独关闭。

关闭账号 A：

```powershell
$env:WEINSIGHT_MYSQL_PASSWORD='weinsight_dev'
python -m app.main article-account-disable --config config/config.dev.yaml --account-name "授权公众号名称A"
python -m app.main article-account-list --config config/config.dev.yaml
```

关闭账号 B：


## 5. 扩容后核验

配置 3 个账号后，立即执行：

```powershell
$env:WEINSIGHT_MYSQL_PASSWORD='weinsight_dev'
python -m app.main article-account-list --config config/config.dev.yaml
python -m app.main article-runtime-metrics --config config/config.dev.yaml --hours 24
python -m app.main trial-monitor-report --config config/config.dev.yaml --hours 24
```

核验项：

```text
3 个账号均可见
3 个账号均为启用状态
优先级分别为 1、2、3
每个账号 poll_interval_minutes 为 60
每个账号 max_articles_per_round 为 3
collect_today_only 仍然生效
article 链路没有异常积压
核心群等待未超过阈值
```

## 6. 暂停和回滚

立即暂停 article 链路：

```text
任一账号连续失败 3 次
核心群等待超过阈值
article 链路长时间占用微信 UI
UI 锁超时持续出现
AI 不再是 dry-run
model_called 不等于 0
CLI 或日志出现文章全文、HTML 或具体文章链接
```

回滚到单账号模式：

```text
保留优先级最高且通过验收的 1 个账号
关闭其余账号
重新执行 article-account-list
重新执行 article-runtime-metrics
确认群链路可继续运行
```

只保留群链路：

```text
关闭全部 article 账号
确认 article 链路不再占用微信 UI
继续观察核心群轮询
保留日志、截图和失败任务状态
```

## 7. Go / Watch / No-Go

Go：

```text
Task 67 为 Go 结论
3 个实际授权公众号/订阅号配置完成
article-account-list 核验通过
任一账号失败都可单独关闭
每小时最多执行 1 次
每轮最多 3 篇
只采集当天发布数据
AI 继续 dry-run 且 model_called=0
```

Watch：

```text
3 个账号配置完成，但某一账号需要继续观察
账号失败可定位并可单独关闭
群链路未受明显影响
需要补充 3 账号小窗口运行证据
```

No-Go：

```text
Task 67 不是 Go 结论
无法确认 3 个账号均已授权
任一账号连续失败 3 次
核心群等待超过阈值
无法单独关闭失败账号
AI 不再是 dry-run
用户可见输出出现敏感内容
```
