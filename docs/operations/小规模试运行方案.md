# 小规模试运行方案

本文档用于第七阶段无 AI 小规模试运行。目标是在不扩大风险面的前提下，验证双链路稳定性、日报质量和人工巡检节奏。

## 1. 试运行规模

硬性规模：

```text
核心群不超过5个
公众号/订阅号不超过20个
开发阶段暂时不注册 Windows 计划任务
```

试运行期间继续使用同一台主机和同一个微信 PC UI 的串行交错模型。所有涉及微信窗口的操作必须通过 `wechat_ui_lock` 互斥执行；巡检、日报查看、日报导出和质量评估不占用微信 UI。

## 2. 试运行前置条件

开始前确认：

```text
微信 PC 4.1.8.107 已登录
微信自动更新已关闭
核心群清单已确认
公众号/订阅号清单已确认
MySQL 开发库可访问
群日报、文章日报、汇总日报命令可手动执行
```

## 3. 每日执行节奏

上午启动前：

```powershell
python -m app.main wechat-health --config config/config.dev.yaml
python -m app.main group-runtime-summary --config config/config.dev.yaml --limit 5
python -m app.main article-runtime-metrics --config config/config.dev.yaml --hours 24
```

中午巡检：

```powershell
python -m app.main group-runtime-metrics --config config/config.dev.yaml --hours 12
python -m app.main article-runtime-metrics --config config/config.dev.yaml --hours 12
python -m app.main group-task-failed-list --config config/config.dev.yaml --limit 20
python -m app.main article-task-failed-list --config config/config.dev.yaml --limit 20
```

晚上日报检查：

```powershell
python -m app.main group-daily-report-list --config config/config.dev.yaml --date 2026-07-07 --limit 20
python -m app.main article-daily-report-list --config config/config.dev.yaml --date 2026-07-07 --limit 20
python -m app.main summary-daily-report-show --config config/config.dev.yaml --date 2026-07-07
python -m app.main summary-daily-report-export --config config/config.dev.yaml --date 2026-07-07 --output runtime/reports/summary
```

## 4. 人工巡检点

每次巡检记录：

```text
群链路成功数、失败数、积压数
article 链路成功数、失败数、积压数
UI 锁等待、持有和超时情况
日报是否按日期生成
汇总日报是否可读
失败任务是否集中在同一群或同一账号
```

## 5. 暂停条件

出现以下任一情况，暂停本轮试运行：

```text
核心群等待超过阈值
任一账号连续失败 3 次
微信掉线、锁屏或窗口卡死
UI 锁长时间未释放
群链路任务异常积压
article 链路任务异常积压
CLI 输出出现正文或具体文章链接
日报内容出现明显不可用或误导性结论
```

暂停后优先恢复群链路，再评估是否重新开启 article 链路。

## 6. 回滚入口

关闭 article 链路：

```powershell
python -m app.main article-account-disable --config config/config.dev.yaml --account-name "授权公众号名称"
```

只保留群链路巡检：

```powershell
python -m app.main group-runtime-summary --config config/config.dev.yaml --limit 5
python -m app.main group-runtime-metrics --config config/config.dev.yaml --hours 24
```

回滚后继续观察：

```text
微信群链路是否恢复
UI 锁是否释放
失败任务是否停止增加
日报和汇总日报是否仍能只读查看
```

## 7. 通过标准

连续试运行通过条件：

```text
核心群采集未被 article 链路长时间阻塞
群链路和 article 链路失败可定位
失败任务可查看且可限量重试
群日报、文章日报和汇总日报均可生成
CLI 输出未出现正文或具体文章链接
人工巡检能够支持 Go / No-Go 判断
```
