# 第九阶段双链路交错真实小窗口验证记录

> **公众号 article-RPA 历史记录（已退役）**：本文中的 `article-rpa-probe`、`collect-article-once`、`run-article-scheduler` 等命令已移除，**禁止执行**，不得作为现行操作指引。公众号现行采集路径仅为 **WeRSS + WeInsight RSS**；请使用 `docs/operations/公众号RSS采集运行手册.md`。

本文档用于第九阶段 Task 66。目标是在 Task 64 单公众号/订阅号真实 POC 执行和 Task 65 单核心群真实轮询 POC 执行均通过后，用 1 到 2 小时的小窗口验证双链路真实交错执行能力。

## 1. 前置条件

必须先完成：

```text
Task 63：真实 POC 环境核验结论为 Go
Task 64：单公众号/订阅号真实 POC 执行结论为 Go 或 Watch
Task 65：单核心群真实轮询 POC 执行结论为 Go 或 Watch
AI 继续 dry-run
model_called=0
```

本轮验证只允许：

```text
1 个实际授权公众号/订阅号
1 个实际授权核心群
1 到 2 小时
手动命令触发
有人值守
不注册 Windows 计划任务
不启用无人值守
```

不允许：

```text
扩到 3 个账号
注册 Windows 计划任务
启动无人值守
启动长期后台调度
绕过 wechat_ui_lock 操作微信 UI
输出聊天正文、文章全文、HTML 或具体文章链接
```

## 2. 验证原则

硬性原则：

```text
群链路优先
article 链路可中断
article 链路不得长时间占用微信 UI
核心群等待超过阈值立即暂停 article 链路
所有微信 UI 操作必须通过 wechat_ui_lock
同一时刻只能有一个链路持有 wechat_ui_lock
```

说明：

```text
双链路交错不是 UI 并行
同一微信 PC UI 只能串行交错执行
article 链路被核心群打断是预期行为
被打断后必须能继续或回滚到单账号模式
核心群优先级高于 article 链路
```

## 8. 暂停和回滚

立即暂停 article 链路：

```text
核心群等待超过阈值立即暂停 article 链路
article 链路不得长时间占用微信 UI
article 链路连续失败 3 次
article 链路 UI 锁超时
AI 不再是 dry-run
model_called 不等于 0
CLI 或日志出现聊天正文、文章全文、HTML 或具体文章链接
```

关闭 article 链路：

```powershell
$env:WEINSIGHT_MYSQL_PASSWORD='weinsight_dev'
python -m app.main article-account-disable --config config/config.dev.yaml --account-name "授权公众号名称"
python -m app.main article-runtime-metrics --config config/config.dev.yaml --hours 1
```

只保留群链路：

```powershell
$env:WEINSIGHT_MYSQL_PASSWORD='weinsight_dev'
python -m app.main group-runtime-summary --config config/config.dev.yaml --limit 5
python -m app.main group-runtime-metrics --config config/config.dev.yaml --hours 1
python -m app.main run-group-scheduler --once --config config/config.dev.yaml
```

回滚到单账号模式：

```text
停止双链路交错验证
只保留 1 个实际授权公众号/订阅号
只使用 collect-article-once 和 parse-article-once
重新执行 Task 64 记录模板
```

## 9. Go / Watch / No-Go

Go：

```text
1 到 2 小时内群链路优先保持有效
核心群等待未超过阈值
article 链路可中断且可恢复
wechat_ui_lock 未出现长时间占用
group/article 指标可解释
AI 继续 dry-run 且 model_called=0
```

Watch：

```text
article 链路有可定位失败，但未影响核心群
UI 锁短暂等待但可恢复
失败任务可查看和限量重试
需要延长 1 到 2 小时观察
```

No-Go：

```text
核心群等待超过阈值
article 链路不可中断
article 链路长时间占用微信 UI
UI 锁超时持续出现
关闭 article 链路后群链路仍异常
AI 不再是 dry-run
用户可见输出出现敏感内容
```
