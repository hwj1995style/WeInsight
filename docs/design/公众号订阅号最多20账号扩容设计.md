# 公众号订阅号最多20账号扩容设计

本文档用于承接四账号真实 POC 与 1-2 小时小窗口稳定性验证后的扩容设计。当前结论是：允许进入最多 20 个账号扩容设计；不允许直接一次性启用 20 个账号。

## 1. 背景与现状

已完成的验证：

```text
4 个账号 controlled POC：Go
4 个账号 scheduler-once 小步验证：Go
两轮自然 due 小窗口验证：Go
核心群补充闭环：Go
article_backlog_count=0
group_backlog_count=0
ui_lock_timeout_count=0
```

已验证账号：

```text
一箱蛋
贵阳鸡蛋价格
家美鲜鸡蛋 佳美鲜
河北馆陶鸡蛋报价
```

已知历史噪声：

```text
24 小时 article_failed_count=1 为家美鲜鸡蛋 佳美鲜 修复前历史失败日志，不代表当前待处理任务。
```

## 2. 目标与非目标

目标：

```text
设计最多 20 个公众号/订阅号的小规模扩容路径。
保持 group/article 双链路共享微信 UI 的互斥安全。
保持每次 UI 锁只处理 1 个 article 账号。
保证任一账号失败可单独关闭，不影响核心群和其他账号。
继续不输出真实文章链接、正文和聊天内容。
```

非目标：

```text
不在本文档中直接启用 20 个账号。
不绕过 poll_interval_minutes。
不把 max_accounts_per_ui_slice 从 1 提高到大于 1。
不注册无人值守 Windows 计划任务。
不启用外部 AI 模型真实调用。
不新增真实文章链接、正文或 HTML 到用户可见文档。
```

## 3. 影响范围

后端影响：

```text
wechat_public_account_config：继续作为账号配置和 due 判断来源。
wechat_article_route_cache：继续作为账号取链路径缓存。
wechat_article_collect_log：继续记录采集结果、失败原因和安全计数。
wechat_article_collect_progress：继续记录中断恢复状态。
wechat_article_raw / clean / analysis / daily_report：保持现有后续链路不变。
wechat_ui_lock：继续串行化 group/article 对真实微信 UI 的访问。
```

前端影响：

```text
无前端页面改动。
只通过 CLI、文档记录和数据库指标推进。
```

运行影响：

```text
真实微信 UI 会被 article 账号逐个打开和关闭。
微信内置浏览器和公众号单聊窗口必须在取链后关闭。
核心群链路仍保持优先级，article 链路可中断、可恢复。
```

## 4. 数据模型与权限模型

不新增数据库表。

继续复用：

```text
wechat_public_account_config.enabled：单账号开关。
wechat_public_account_config.priority：账号分批顺序。
wechat_public_account_config.poll_interval_minutes：账号级节流。
wechat_public_account_config.max_articles_per_round：单轮文章数，扩容阶段固定为 1。
wechat_public_account_config.last_success_collect_time：scheduler due 判断。
wechat_article_route_cache.status：取链路径缓存状态。
```

安全边界：

```text
数据库可保存 article_url 供内部解析使用。
用户可见文档、CLI 汇总和验收记录不得输出真实文章链接。
剪贴板只允许短暂读取并恢复。
失败截图只用于本机诊断，不进入用户可见报告。
```

## 5. 后端实现方案

### 5.1 扩容批次

采用分批扩容，不一次性启用 20 个：

```text
Batch 0：保留当前 4 个账号，继续观察。
Batch 1：扩到 8 个账号，新增 4 个。
Batch 2：扩到 12 个账号，新增 4 个。
Batch 3：扩到 16 个账号，新增 4 个。
Batch 4：扩到最多 20 个账号，新增最多 4 个。
```

每个批次必须完成：

```text
至少 2 轮自然 due。
每轮每个启用账号 link_count >= 1 或明确判定为当天无新文。
failed_count=0。
lock_timeout_count=0。
article_backlog_count=0。
group_backlog_count=0。
核心群可在同一窗口内成功采集并完成 clean/analyze/report。
```

### 5.2 调度策略

扩容阶段保持：

```text
max_accounts_per_ui_slice=1
max_articles_per_round=1
poll_interval_minutes=60
daily_window_start=07:30
daily_window_end=19:30
collect_today_only=1
```

优先级建议：

```text
当前 4 个账号保持 priority=1..4。
新增账号按批次追加 priority。
同批账号按业务重要性排序。
```

单轮 UI 预算估算：

```text
每个账号按 15 到 30 秒估算。
20 个账号每小时 article UI 占用约 5 到 10 分钟。
核心群仍可在 article 分片之间插入执行。
```

### 5.3 取链策略

继续使用既有策略：

```text
优先读取 route cache。
缓存失败达到阈值后重新 probe。
probe 顺序覆盖底部菜单、历史消息、公众号主页、聊天文章卡片。
成功后更新 route cache。
取链后关闭微信内置浏览器和公众号单聊窗口。
```

新增账号准入前必须完成：

```text
article-rpa-probe status=ok。
link_count>=1。
browser_window_present=0。
account_window_present=0。
不写数据库。
不输出真实链接。
```

### 5.4 失败隔离与回滚

单账号回滚：

```text
任一账号连续失败 2 次：标记 Watch，暂停该账号，保留其他账号。
任一账号连续失败 3 次：禁用该账号，记录 No-Go 原因。
```

批次回滚：

```text
任一批次出现 ui_lock_timeout：停止继续扩容。
article_backlog_count 或 group_backlog_count 未归零：停止继续扩容。
核心群失败：停止 article 扩容，优先恢复群链路。
日报质量明显下降：停止继续扩容，回到上一批次账号数。
```

全局回滚：

```text
关闭新增批次账号 enabled。
保留当前已验证的 4 个账号。
必要时只保留群链路。
```

## 7. 风险与分阶段落地建议

主要风险：

```text
不同公众号历史文章入口差异继续扩大。
新增账号可能没有当天正文文章。
微信 UI 控件结构变化导致取链路径失效。
20 个账号导致单机 UI 占用时间过长。
浏览器或公众号窗口未关闭造成后续取错账号链接。
```

缓解措施：

```text
新增账号先 probe，不直接 collect。
每批最多新增 4 个账号。
每个账号仍保持 max_articles_per_round=1。
连续失败账号单独禁用。
每轮后检查窗口清理、runtime metrics、failed task 和 backlog。
```

推荐落地顺序：

```text
1. 准备候选账号清单，不超过 20 个。
2. 对候选账号逐个执行 RPA-only probe。
3. 只把 probe 通过账号加入下一批配置。
4. 每批新增最多 4 个账号。
5. 每批完成两轮自然 due 和核心群闭环后再进入下一批。
6. 达到 20 个后继续人工值守，不立即无人值守。
```

## 12. 2026-07-09 全账号无当日正文 no-data 流程门禁

目标语义：

```text
当天无正文发布不是采集失败，而是所有公众号账号的正常 no-data 结果；上海只是本轮真实验证样本，不是账号特例。
RPA 只允许检查第一条可见历史文章；如果列表首条没有目标日期，不允许下探第二条或更低行，而是点击第一条正文读取详情页发布时间；详情页也不是目标日期时，本轮停止，不复制链接、不写 raw、不创建 clean/analyze/report 任务。
调度层应将 link_count=0 的正常空结果记录为 skipped/no-data，而不是 failed。
no-data 结果需要更新本轮账号采集进度，避免 scheduler 在同一轮反复重试该账号。
```

通用验证目标：

```text
任意公众号账号在目标日期无文章时，都允许运行受控 collect 并正常结束；上海禽蛋价格综合报价作为 2026-07-09 的实测样本。

后处理预期：parse/analyze/daily_report 无新增任务，article-task-failed-list 仍为空，article-runtime-metrics 无 backlog。
```

## 13. 2026-07-09 Batch 1 通用 no-data 受控小窗口
